왜 크기가 필요한가

Rust 컴파일러는 모든 값의 크기를 컴파일 타임에 알아야 합니다.

크기 문제 해결 방법개념
// ❌ 불가: 트레이트 객체의 크기를 알 수 없음
fn bad(animal: Animal) { ... }

// ✅ 해결책:
fn ok1(animal: &dyn Animal) { ... }  // Fat Ptr — 항상 16 bytes
fn ok2(animal: Box<dyn Animal>) { ... }  // Fat Ptr — 항상 16 bytes
fn ok3(animal: impl Animal) { ... }  // 단형화 — 컴파일 타임 확정

dyn-compatible 규칙 (Rust Reference)

dyn-compatible 조건✅ 스펙 보장
✅ dyn-compatible 조건 (모두 만족해야 함):
  1. 메서드의 반환 타입이 Self가 아님
  2. 메서드가 제네릭 타입 파라미터를 사용하지 않음
  3. 메서드가 Self: Sized 바운드를 갖지 않음

❌ dyn-incompatible 예시:
  trait Bad {
      fn clone(&self) -> Self;  // Self 반환 — 불가
      fn compare<T>(&self, other: T);  // 제네릭 — 불가
  }
  // Box<dyn Bad> 사용 불가!

✅ 언어 스펙으로 보장:
  &dyn Trait 크기 = 16 bytes (64bit 시스템)

⚠️ 구현 세부사항 (비보장):

❌ 절대 금지:
  unsafe transmute로 vtable 직접 조작 → UB
  std::raw::TraitObject (Rust 1.72 제거됨)

#1 트레이트 정의와 구현 — 가장 기본
입문
RUST 
trait Animal {
    fn speak(&self) -> String;        // 구현 강제
    fn name(&self)  -> &str;
    fn introduce(&self) -> String {   // 기본 구현 — 오버라이드 가능
        format!("저는 {}이고 {}라고 합니다.", self.name(), self.speak())
    }
}

struct Lion { name: String }
struct Duck { name: String }

impl Animal for Lion {
    fn speak(&self) -> String { "Roar!".to_string() }
    fn name(&self)  -> &str   { &self.name }
}
impl Animal for Duck {
    fn speak(&self) -> String { "Quack!".to_string() }
    fn name(&self)  -> &str   { &self.name }
    fn introduce(&self) -> String {
        format!("꽥꽥! 저는 {}입니다.", self.name)
    }
}

fn main() {
    let lion = Lion { name: "Simba".to_string() };
    let duck = Duck { name: "Donald".to_string() };
    println!("{}", lion.speak());      // Roar!
    println!("{}", duck.speak());      // Quack!
    println!("{}", lion.introduce());  // 저는 Simba이고 Roar!라고 합니다.
    println!("{}", duck.introduce());  // 꽥꽥! 저는 Donald입니다.
}

트레이트(trait)는 공통 인터페이스를 정의하는 Rust의 핵심 도구입니다. 메서드 선언으로 구현을 강제하고, 기본 구현으로 공통 동작을 제공합니다.

기본 메서드도 vtable에 포함됩니다. Lion의 introduce() 슬롯에는 기본 구현 주소가, Duck의 슬롯에는 오버라이드된 주소가 들어갑니다.

💡
출처: Rust Book Ch10.2 — Traits
#2 &dyn Trait — 소유권 없이 다형성
입문
RUST 
trait Animal {
    fn speak(&self) -> String;
    fn name(&self)  -> &str;
}
struct Lion { name: String }
struct Duck { name: String }
impl Animal for Lion {
    fn speak(&self) -> String { format!("{}: Roar!", self.name) }
    fn name(&self)  -> &str { &self.name }
}
impl Animal for Duck {
    fn speak(&self) -> String { format!("{}: Quack!", self.name) }
    fn name(&self)  -> &str { &self.name }
}

// &dyn Animal: Fat Pointer(16 bytes), 힙 할당 없음
fn print_animal(a: &dyn Animal) {
    println!("[{}] {}", a.name(), a.speak());
}

fn main() {
    let lion = Lion { name: "Simba".to_string() };
    let duck = Duck { name: "Donald".to_string() };
    print_animal(&lion);
    print_animal(&duck);
    // 소유권 이동 없음 — 여전히 사용 가능
    println!("크기: {} bytes",
        std::mem::size_of::<&dyn Animal>()); // 16
}

&dyn Animal은 Fat Pointer입니다. 일반 참조(&Lion, 8 bytes)와 달리 16 bytes입니다.

  • data_ptr (8 bytes) — 실제 데이터 주소
  • vtable_ptr (8 bytes) — 함수 포인터 테이블 주소
✅
&dyn Trait 크기 = 16 bytes는 언어 스펙으로 보장됩니다. — Rust Reference
#3 Box<dyn Trait> — 힙에 올려 소유권 이전
입문
RUST 
use std::mem;
trait Animal { fn speak(&self) -> String; }
struct Lion { name: String }
struct Duck { name: String }
impl Animal for Lion { fn speak(&self) -> String { format!("{}: Roar!", self.name) } }
impl Animal for Duck { fn speak(&self) -> String { format!("{}: Quack!", self.name) } }

fn make_lion(name: &str) -> Box<dyn Animal> {
    Box::new(Lion { name: name.to_string() })
    // 1. Lion을 힙에 할당
    // 2. data_ptr + vtable_ptr = Fat Pointer 반환
}
fn make_duck(name: &str) -> Box<dyn Animal> {
    Box::new(Duck { name: name.to_string() })
}

fn main() {
    let a = make_lion("Simba");
    let b = make_duck("Donald");
    println!("{}", a.speak());
    println!("{}", b.speak());

    // 크기 비교
    println!("Box<dyn Animal>: {} bytes (fat)",  mem::size_of::<Box<dyn Animal>>());
    println!("Box<Lion>:       {} bytes (thin)", mem::size_of::<Box<Lion>>());
} // a, b 자동 Drop (vtable[0] drop_in_place 경유)

Box<dyn Animal>는 힙에 데이터를 저장하고 그 소유권을 가집니다. 함수에서 반환하거나 구조체 필드에 저장할 수 있습니다.

Box<Lion>은 8 bytes(thin pointer), Box<dyn Animal>은 vtable_ptr도 포함해 16 bytes입니다.

⚠️
힙 할당 비용이 발생합니다. 성능이 중요한 루프에서는 impl Trait나 enum을 고려하세요.
#4 Vec<Box<dyn Trait>> — 여러 타입을 하나의 컬렉션에
입문
RUST 
trait Animal {
    fn speak(&self) -> String;
    fn name(&self)  -> &str;
    fn legs(&self)  -> u32;
}
struct Lion  { name: String }
struct Duck  { name: String }
struct Snake { name: String }
impl Animal for Lion  {
    fn speak(&self)->String{format!("{}: Roar!",self.name)}
    fn name(&self)->&str{&self.name} fn legs(&self)->u32{4}
}
impl Animal for Duck  {
    fn speak(&self)->String{format!("{}: Quack!",self.name)}
    fn name(&self)->&str{&self.name} fn legs(&self)->u32{2}
}
impl Animal for Snake {
    fn speak(&self)->String{format!("{}: Hiss!",self.name)}
    fn name(&self)->&str{&self.name} fn legs(&self)->u32{0}
}

fn main() {
    let zoo: Vec<Box<dyn Animal>> = vec![
        Box::new(Lion  { name: "Simba".to_string() }),
        Box::new(Duck  { name: "Donald".to_string() }),
        Box::new(Snake { name: "Kaa".to_string() }),
    ];
    for a in &zoo { println!("{}", a.speak()); }

    let total: u32 = zoo.iter().map(|a| a.legs()).sum();
    println!("총 동물: {}마리, 총 다리: {}개", zoo.len(), total);
}

Vec<Box<dyn Animal>>은 이형 컬렉션입니다. Vec의 각 원소는 16 bytes Fat Pointer이며, 실제 데이터는 각각 별도의 힙 위치에 있습니다.

같은 타입(Lion)은 같은 vtable을 공유합니다 — vtable은 타입당 1개만 존재합니다.

💡
타입이 고정되어 있다면 enum이 더 효율적입니다(힙 할당 없음). 예제 #15에서 비교합니다.
#5 조건 분기 반환 — impl Trait으로 불가능한 것
입문
RUST 
trait Animal { fn speak(&self)->&str; fn kind(&self)->&str; }
struct Lion; struct Duck;
impl Animal for Lion { fn speak(&self)->&str{"Roar!"} fn kind(&self)->&str{"Lion"} }
impl Animal for Duck { fn speak(&self)->&str{"Quack!"} fn kind(&self)->&str{"Duck"} }

// ❌ impl Trait으로는 불가능
// fn make(kind: &str) -> impl Animal {
//     if kind == "lion" { Lion } else { Duck }
//     // ERROR: `if` and `else` have incompatible types
// }

// ✅ Box<dyn Trait>으로 가능
fn make_animal(kind: &str) -> Box<dyn Animal> {
    match kind {
        "lion" => Box::new(Lion),
        "duck" => Box::new(Duck),
        other  => panic!("알 수 없는 종류: {}", other),
    }
    // Box<dyn Animal>은 항상 16 bytes — 내용 무관!
}

fn main() {
    for kind in &["lion", "duck", "lion"] {
        let a = make_animal(kind);
        println!("[{}] {}", a.kind(), a.speak());
    }
}

Rust Reference 공식: "Each possible return value from the function must resolve to the same concrete type."

impl Trait은 모든 분기에서 동일한 구체 타입을 반환해야 합니다. Box<dyn Animal>의 크기는 항상 16 bytes이므로 컴파일러가 허용합니다.

⚠️
실무에서는 panic! 대신 Result<Box<dyn Animal>, String>을 반환합니다 (예제 #9 참고).
#6 구조체 필드에 Box<dyn Trait> 저장 — Zoo 패턴
입문
RUST 
trait Animal { fn speak(&self)->String; fn name(&self)->&str; }

impl Animal for Lion { fn speak(&self)->String{format!("{}: Roar!",self.name)} fn name(&self)->&str{&self.name} }

    name:    String,
    animals: Vec<Box<dyn Animal>>,  // ✅ 구조체 필드로 가능

// struct Bad { animal: impl Animal }  // ❌ 컴파일 오류

impl Zoo {
    fn new(name: &str) -> Self { Zoo { name: name.to_string(), animals: vec![] } }
    fn add(&mut self, a: Box<dyn Animal>) {
        println!("[{}] {} 입장!", self.name, a.name());

    }

    }
}

fn main() {
    let mut zoo = Zoo::new("서울 동물원");
    zoo.add(Box::new(Lion { name: "Simba".to_string() }));
    zoo.add(Box::new(Duck { name: "Donald".to_string() }));

}

컴파일러는 구조체 크기를 컴파일 타임에 확정해야 합니다. impl Animal은 어떤 타입인지 모르므로 크기를 알 수 없어 필드로 사용 불가입니다. Box<dyn Animal>은 항상 16 bytes이므로 가능합니다.

💡
제네릭 구조체(struct Zoo<T: Animal>)는 하나의 타입만 담을 수 있습니다. 여러 타입을 담으려면 Box<dyn>이 필요합니다.
#7 Option<Box<dyn Trait>> — null-safe 다형성
입문
RUST 
trait Animal { fn speak(&self)->&str; }
struct Lion; struct Duck; struct Eagle;
impl Animal for Lion  { fn speak(&self)->&str{"Roar!"}    }
impl Animal for Duck  { fn speak(&self)->&str{"Quack!"}   }
impl Animal for Eagle { fn speak(&self)->&str{"Screech!"} }

fn find(name: &str) -> Option<Box<dyn Animal>> {
    match name {
        "lion"  => Some(Box::new(Lion)),
        "duck"  => Some(Box::new(Duck)),
        "eagle" => Some(Box::new(Eagle)),
        _       => None,
    }
}

fn main() {
    for name in &["lion","duck","whale","eagle"] {
        match find(name) {
            Some(a) => println!("{}: {}", name, a.speak()),
            None    => println!("{}: (없음)", name),
        }
    }
    // Option 메서드 활용
    let sound = find("lion")
        .map(|a| a.speak().to_string())
        .unwrap_or("?".to_string());
    println!("사자 소리: {}", sound);

    // null pointer optimization: size_of::<Option<Box<dyn Animal>>>() == 16
    use std::mem;
    println!("Option<Box<dyn>> 크기: {} bytes",
        mem::size_of::<Option<Box<dyn Animal>>>());
}

Option<Box<dyn Trait>>는 Rust에서 null-safe하게 다형적 값을 표현합니다. Rust 컴파일러는 Option<Box<T>> 크기를 Box<T>와 동일하게 최적화합니다 (null pointer optimization).

✅
size_of::<Option<Box<dyn T>>>() == 16 bytes — null pointer optimization은 언어 스펙 보장입니다.
#8 dyn-compatible 위반 — Self 반환과 Clone 우회
중급
RUST 
// Clone 트레이트의 clone(&self) -> Self 는 dyn-incompatible
// 해결: helper trait 패턴

trait AnimalClone {
    fn clone_box(&self) -> Box<dyn Animal>; // Self 대신 Box<dyn> 반환
}
impl<T: Animal + Clone + 'static> AnimalClone for T {
    fn clone_box(&self) -> Box<dyn Animal> { Box::new(self.clone()) }
}

trait Animal: AnimalClone {
    fn speak(&self) -> String;
}
impl Clone for Box<dyn Animal> {
    fn clone(&self) -> Box<dyn Animal> { self.clone_box() }
}

#[derive(Clone)] struct Lion { name: String }
#[derive(Clone)] struct Duck { name: String }
impl Animal for Lion { fn speak(&self)->String{format!("{}: Roar!",self.name)} }
impl Animal for Duck { fn speak(&self)->String{format!("{}: Quack!",self.name)} }

fn main() {
    let original: Vec<Box<dyn Animal>> = vec![
        Box::new(Lion { name: "Simba".to_string() }),
        Box::new(Duck { name: "Donald".to_string() }),
    ];
    let cloned = original.clone(); // Box<dyn Animal> 복제!
    println!("원본:"); for a in &original { println!("  {}", a.speak()); }
    println!("복제:"); for a in &cloned   { println!("  {}", a.speak()); }
}

Clone의 clone(&self) -> Self는 dyn-compatible하지 않습니다. vtable에 가변 크기 반환 슬롯을 만들 수 없기 때문입니다.

해결책: 반환 타입을 Box<dyn Animal>(고정 16 bytes)로 만든 helper trait을 중간에 두는 방법입니다.

✅
이 패턴은 Rust 커뮤니티 공인 방법입니다. Rust Reference dyn-compatibility 공식 문서에서도 언급합니다.

💡

#10 전략(Strategy) 패턴 — 런타임 알고리즘 교체
중급
RUST 
trait MoveStrategy {
    fn do_move(&self, name: &str);
    fn strategy_name(&self) -> &str;
}
struct Walk; struct Fly; struct Swim;
impl MoveStrategy for Walk { fn do_move(&self,n:&str){println!("{}: 터벅터벅",n)} fn strategy_name(&self)->&str{"Walk"} }
impl MoveStrategy for Fly  { fn do_move(&self,n:&str){println!("{}: 훨훨!",n)}    fn strategy_name(&self)->&str{"Fly"}  }
impl MoveStrategy for Swim { fn do_move(&self,n:&str){println!("{}: 첨벙~",n)}    fn strategy_name(&self)->&str{"Swim"} }

struct Animal {
    name:     String,
    strategy: Box<dyn MoveStrategy>,  // 런타임 교체 가능!
}
impl Animal {
    fn new(name: &str, s: Box<dyn MoveStrategy>) -> Self {
        Animal { name: name.to_string(), strategy: s }
    }
    fn act(&self) {
        print!("[전략: {}] ", self.strategy.strategy_name());
        self.strategy.do_move(&self.name);
    }
    fn set_strategy(&mut self, s: Box<dyn MoveStrategy>) {
        println!("→ 전략 변경: {} → {}", self.strategy.strategy_name(), s.strategy_name());
        self.strategy = s; // 이전 Box 자동 Drop
    }
}

fn main() {
    let mut donald = Animal::new("Donald", Box::new(Fly));
    donald.act();
    donald.set_strategy(Box::new(Swim));
    donald.act();
    donald.set_strategy(Box::new(Walk));
    donald.act();
}

self.strategy = s; 한 줄로 전략이 교체됩니다. 이전 Box는 자동 Drop(vtable[0] drop_in_place 경유)되고 새 전략이 저장됩니다.

💡
컴파일 타임 전략 고정이 필요하면 제네릭(struct Animal<S: MoveStrategy>)을 사용하세요. 런타임 교체가 필요할 때만 Box<dyn>을 씁니다.

trait Observer { fn on_event(&self, e: &str); fn label(&self) -> &str; }

        println!("[Bus] '{}' 구독", o.label());
        self.observers.push(o);
    }
    fn publish(&self, event: &str) {
        println!("\n--- 이벤트: '{}' ---", event);
        for o in &self.observers { o.on_event(event); }
    }
}

    bus.publish("animal_born");
    bus.publish("animal_fed");
    bus.publish("animal_died");
}

    }
    fn run(&self, name:&str, input:&str) -> Result<String, String> {

#심화19 커맨드 패턴 — Undo/Redo

trait Command { fn exec(&self, zoo:&mut Vec<String>); fn undo(&self, zoo:&mut Vec<String>); fn name(&self)->&str; }

struct AddAnimal    { name: String }
struct RemoveAnimal { name: String }

impl Command for AddAnimal {
    fn name(&self)->&str{"Add"}
    fn exec(&self,zoo:&mut Vec<String>){ zoo.push(self.name.clone()); println!("  + '{}'",self.name); }
    fn undo(&self,zoo:&mut Vec<String>){ zoo.retain(|n|n!=&self.name); println!("  - '{}' 취소",self.name); }
}
impl Command for RemoveAnimal {
    fn name(&self)->&str{"Remove"}
    fn exec(&self,zoo:&mut Vec<String>){ zoo.retain(|n|n!=&self.name); println!("  - '{}'",self.name); }
    fn undo(&self,zoo:&mut Vec<String>){ zoo.push(self.name.clone()); println!("  + '{}' 복원",self.name); }
}

struct History { zoo: Vec<String>, history: Vec<Box<dyn Command>> }
impl History {
    fn new()->Self{History{zoo:vec![],history:vec![]}}
    fn exec(&mut self, cmd: Box<dyn Command>) {
        println!("[{}]", cmd.name()); cmd.exec(&mut self.zoo);
        self.history.push(cmd);
        println!("  상태: {:?}", self.zoo);
    }
    fn undo(&mut self) {
        if let Some(cmd)=self.history.pop(){
            println!("[Undo: {}]", cmd.name()); cmd.undo(&mut self.zoo);
            println!("  상태: {:?}", self.zoo);
        }
    }
}

fn main() {

    h.exec(Box::new(AddAnimal    { name: "Lion".into() }));
    h.exec(Box::new(AddAnimal    { name: "Duck".into() }));
    h.exec(Box::new(RemoveAnimal { name: "Lion".into() }));
    println!("\n=== Undo 2회 ===");
    h.undo(); h.undo();
}

            print!("  [{}] ",m.name());

            println!("통과");

        Response{status:200,body:format!("OK: {}",req.path)}

}

type Handler = Box<dyn Fn(&str) -> Option<String>>;

fn make_validator(min: usize) -> Handler {
    Box::new(move |s: &str| {
        if s.len() >= min { None }
        else { Some(format!("너무 짧음: {} < {}", s.len(), min)) }
    })
}
fn make_transformer(prefix: &'static str) -> Handler {
    Box::new(move |s: &str| Some(format!("{}{}", prefix, s.to_uppercase())))
}
fn make_filter(bad: &'static str) -> Handler {
    Box::new(move |s: &str| {
        if s.contains(bad) { Some(format!("금지어: '{}'", bad)) }
        else { None }
    })
}

struct Pipeline { steps: Vec<Handler> }
impl Pipeline {
    fn new() -> Self { Pipeline { steps: vec![] } }
    fn add(mut self, h: Handler) -> Self { self.steps.push(h); self }
    fn run(&self, input: &str) -> String {
        let mut data = input.to_string();
        for step in &self.steps {
            if let Some(result) = step(&data) {
                data = result;
            }
        }
        data
    }
}

fn main() {
    let p = Pipeline::new()
        .add(make_validator(3))
        .add(make_filter("BAD"))
        .add(make_transformer("→ "));
    for s in &["hello","ok","BAD word","world"] {
        println!("'{}' → '{}'", s, p.run(s));
    }
}

    for hh in h { hh.join().unwrap(); }
    println!("ref count: {}", Arc::strong_count(&lion));
}

Arc::clone()은 실제 데이터나 vtable을 복사하지 않고 참조 카운트만 증가시킵니다. 모든 클론이 같은 vtable을 공유합니다.

💡
Arc<dyn T>는 T: Send + Sync가 필요합니다.
#9 다운캐스트 — Any 트레이트
고급
RUST 
use std::any::Any;
trait Animal: Any { fn speak(&self) -> &str; fn as_any(&self) -> &dyn Any; }
struct Lion { pub age: u32 } struct Duck { pub can_fly: bool }
impl Animal for Lion  { fn speak(&self)->&str{"Roar"}  fn as_any(&self)->&dyn Any{self} }
impl Animal for Duck  { fn speak(&self)->&str{"Quack"} fn as_any(&self)->&dyn Any{self} }

fn describe(a: &dyn Animal) {
    println!("소리: {}", a.speak());
    if let Some(l) = a.as_any().downcast_ref::<Lion>() { println!("  Lion! 나이={}", l.age); }
    if let Some(d) = a.as_any().downcast_ref::<Duck>() { println!("  Duck! 비행={}", d.can_fly); }
}
fn main() {
    let zoo: Vec<Box<dyn Animal>> = vec![Box::new(Lion{age:5}), Box::new(Duck{can_fly:true})];
    for a in &zoo { describe(a.as_ref()); println!(); }
}

다운캐스트는 dyn Trait에서 구체 타입을 복원합니다. downcast_ref::<T>()는 타입 ID를 확인하여 맞으면 Some(&T), 아니면 None을 반환합니다.

⚠️
다운캐스트가 자주 필요하다면 설계 문제일 수 있습니다. 트레이트에 메서드를 추가하거나 enum을 먼저 고려하세요.
#10 종합 — 동적 디스패치 전체 흐름
고급
RUST 
use std::mem;
trait Animal { fn speak(&self)->String; fn legs(&self)->u32; }
struct Lion { name: String } struct Duck { name: String }
impl Animal for Lion { fn speak(&self)->String{format!("{}: Roar!",self.name)} fn legs(&self)->u32{4} }
impl Animal for Duck { fn speak(&self)->String{format!("{}: Quack!",self.name)} fn legs(&self)->u32{2} }

fn main() {
    println!("╔══════════════════════════════════╗");
    println!("║ 동적 디스패치 전체 흐름 종합     ║");
    println!("╚══════════════════════════════════╝\n");

    // 1. 생성 — Box::new() 힙 할당 + Fat Pointer 생성
    let zoo: Vec<Box<dyn Animal>> = vec![
        Box::new(Lion { name: "Simba".into() }),
        Box::new(Duck { name: "Donald".into() }),
    ];

    // 2. Fat Pointer 크기 (스펙 보장)
    println!("[1] Fat Pointer: {} bytes", mem::size_of::<Box<dyn Animal>>());

    // 3. vtable 동일성 확인
    let m1 = std::ptr::metadata(zoo[0].as_ref());
    let m2 = std::ptr::metadata(zoo[1].as_ref());
    println!("[2] Lion·Duck vtable 동일? {}", m1 == m2);

    // 4. vtable[1] 경유 크기
    for a in &zoo {
        println!("[3] size={}B → {}", mem::size_of_val(a.as_ref()), a.speak());
    }

    // 5. vtable[3+] 경유 메서드 호출
    let total: u32 = zoo.iter().map(|a| a.legs()).sum();
    println!("[4] 총 다리: {}개", total);

    println!("\n[5] Drop 시작 (vtable[0] drop_in_place 경유)...");
    // zoo 자동 Drop
}

3부의 모든 내용을 하나로 통합합니다: Box 생성 → Fat Pointer → vtable 확인 → 메서드 호출 → Drop까지 단계별로 보여줍니다.

✅
스펙 보장: 크기=16 bytes, vtable 존재, drop/size/align 슬롯
⚠️
구현 세부사항: vtable 내부 순서, 바이트 오프셋
