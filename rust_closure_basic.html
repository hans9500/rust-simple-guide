§ 01

클로져란 무엇인가 — 기본 개념

클로져(Closure)는 주변 환경(스코프)의 변수를 캡처(capture)할 수 있는 익명 함수입니다. The Rust Book은 클로져를 "anonymous functions you can save in a variable or pass as arguments to other functions"로 정의합니다. 일반 fn과 달리 자신이 정의된 스코프의 변수를 기억합니다.

§ 02

캡처 방식 — 컴파일러 추론 규칙

컴파일러는 클로져 본문에서 변수가 어떻게 사용되는지 분석하여 가장 제한적인(least restrictive) 캡처 방식을 자동으로 선택합니다. 읽기만 하면 불변 참조, 수정하면 가변 참조, 소유권을 이동시키면 소유권 캡처 순서로 결정됩니다. move 키워드를 사용하면 이 추론을 무시하고 모든 변수를 소유권 이전으로 강제합니다.

⚠️ 중요: 캡처 방식 결정 우선순위
컴파일러는 ① 불변 참조(&T) → ② 가변 참조(&mut T) → ③ 소유권 이전(move) 순서로 가능한 가장 약한 방식을 선택합니다. move를 명시하면 Copy 타입은 복사, 나머지는 소유권 이전이 됩니다.

§ 03

클로져 내부 구조 — 컴파일러가 생성하는 익명 구조체

Rust의 클로져는 컴파일 시점에 캡처한 변수들을 필드로 갖는 익명 구조체(anonymous struct)로 변환됩니다. 그리고 그 구조체에 캡처 방식에 따라 Fn, FnMut, FnOnce 중 적절한 트레이트가 자동 구현됩니다. 이 개념을 이해하면 왜 특정 규칙이 존재하는지 명확히 알 수 있습니다.

34struct Cacher<T>
35where
36    T: Fn(u32) -> u32, // T는 Fn 트레이트를 구현해야 함
37{
38    calculation: T,                  // 클로져를 제네릭 필드로 저장
39    value: Option<u32>,            // 계산 결과를 캐시
40}
41
42impl<T> Cacher<T>
43where
44    T: Fn(u32) -> u32,
45{
46    fn new(calculation: T) -> Cacher<T> {
47        Cacher { calculation, value: None }
48    }
49
50    // 처음 호출 시 계산, 이후 캐시된 값 반환 (Lazy evaluation)
51    fn value(&mut self, arg: u32) -> u32 {
52        match self.value {
53            Some(v) => v,
54            None => {
55                let v = (self.calculation)(arg); // 클로져 호출
56                self.value = Some(v);
57                v
58            }
59        }
60    }
61}

트레이트 시그니처 — std::ops 공식 정의

세 트레이트의 핵심 차이는 self 매개변수의 형태에 있습니다. call은 &self, call_mut은 &mut self, call_once는 self를 받습니다. 이 차이가 반복 호출 가능 여부와 원본 변수 수정 가능 여부를 결정합니다.

 7// ① FnOnce — 최상위 트레이트 (모든 클로져가 구현)
 8pub trait FnOnce<Args> {
 9    type Output;
10    // self를 소비 → 딱 한 번만 호출 가능

14// ② FnMut — FnOnce의 서브트레이트
15pub trait FnMut<Args>: FnOnce<Args> {
16    // &mut self → 상태 변경 가능, 반복 호출 가능
17    fn call_mut(&mut self, args: Args) -> Self::Output;
18}

20// ③ Fn — FnMut의 서브트레이트 (가장 엄격한 요구사항)
21pub trait Fn<Args>: FnMut<Args> {
22    // &self → 상태 변경 불가, 반복 호출 가능, 공유 참조로 호출
23    fn call(&self, args: Args) -> Self::Output;
24}

42    let r = &fn_cl; // &F
43    r();            // &F도 Fn 구현 → 호출 가능
44    fn_cl();        // 원본도 그대로

47    let mut n = 0;
48    let mut fnmut_cl = || { n += 1; println!("n = {}", n); };
49    {
50        let rm = &mut fnmut_cl; // &mut F
51        rm();                    // &mut F도 FnMut 구현 → 호출 가능
52    }
53    fnmut_cl(); // n = 2
54}
📌 서브트레이트 관계의 의미
Fn: FnMut는 "Fn은 FnMut의 서브트레이트"를 의미합니다. 즉, Fn을 구현하려면 반드시 FnMut도 구현해야 합니다. 따라서 FnMut를 요구하는 곳에 Fn을 전달할 수 있습니다 (리스코프 치환 원칙과 유사). 하지만 반대로 Fn을 요구하는 곳에 FnMut만 구현한 클로져는 전달할 수 없습니다.

Fn 트레이트 — 예제 5개

Fn은 캡처한 변수를 불변 참조(&T)로만 사용합니다. &self로 호출되므로 몇 번이든 반복 호출 가능하며, 동시에 여러 곳에서 사용할 수도 있습니다. 표준 라이브러리의 map, filter 등 대부분의 Iterator 어댑터가 Fn을 요구합니다.

📌 std::ops::Fn 공식 정의
"An instance of Fn can be called repeatedly without mutating state." — Fn을 구현하는 타입에는 모든 safe 함수 포인터도 포함됩니다. &F가 Fn을 구현하면 &F도 자동으로 FnMut를 구현합니다 (std::ops 문서 명시).

FnMut는 캡처한 변수를 가변 참조(&mut T)로 빌립니다. &mut self로 호출되므로 반복 호출은 가능하지만, 클로져가 살아있는 동안 캡처한 변수를 직접 사용할 수 없습니다. 클로져 변수를 mut로 선언해야 하며, 함수 인자로 받을 때도 mut f: F로 받아야 합니다.

"FnOnce applies to closures that can be called once. All closures implement at least this trait because all closures can be called. A closure that moves captured values out of its body will only implement FnOnce and none of the other Fn traits because it can only be called once."

Box<dyn Fn> vs impl Fn — 동적 vs 정적 디스패치

impl Fn은 컴파일 시점에 구체적인 타입이 결정되는 정적 디스패치(static dispatch)입니다. Box<dyn Fn>은 런타임에 vtable을 통해 호출하는 동적 디스패치(dynamic dispatch)입니다. 서로 다른 타입의 클로져를 같은 컬렉션에 저장하거나 반환 타입이 런타임에 결정될 때는 Box<dyn Fn>을 사용합니다.

✅ impl Fn 사용 시기
함수의 반환 타입이 컴파일 시점에 확정될 때. 성능이 중요할 때. 단일 클로져 반환 시. 제네릭 함수 바운드.
📌 Box<dyn Fn> 사용 시기
서로 다른 클로져를 Vec 등에 저장할 때. 런타임에 클로져 결정 시. 재귀 구조. 트레이트 객체가 필요할 때.

41    // Vec에 서로 다른 클로져 저장 (Box<dyn Fn> 필수)
42    let rules: Vec<Box<dyn Fn(i32) -> bool>> = vec![
43        Box::new(|x| x > 0),
44        Box::new(|x| x % 2 == 0),
45        Box::new(|x| x < 100),
46    ];
47    let n = 42;
48    let all_pass = rules.iter().all(|rule| rule(n));
49    println!("{}: 모든 규칙 통과={}", n, all_pass); // 42: true
50
51    // FnMut, FnOnce도 동일 패턴
52    let mut actions: Vec<Box<dyn FnMut()>> = vec![
53        Box::new(|| println!("Action A")),
54        Box::new(|| println!("Action B")),
55    ];
56    for action in &mut actions { action(); }

Fn &self &T 불변 참조 무제한 불필요 FnMut, FnOnce map, filter, 고차 함수
FnMut &mut self &mut T 가변 참조 무제한 필요 FnOnce sort_by_key, for_each, 카운터
FnOnce self 소비 소유권 이전 단 1회 불필요 — thread::spawn, unwrap_or_else
자동 구현 규칙
Fn ⊆ FnMut ⊆ FnOnce
Fn 구현 → FnMut, FnOnce 자동. FnMut 구현 → FnOnce 자동. 역방향 불성립.
