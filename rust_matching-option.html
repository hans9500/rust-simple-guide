Chapter 1 — Pattern Matching
1-1. match 표현식 — 기본 구조
match는 표현식(expression)입니다. 각 arm은 패턴 => 표현식 형태이며 match 자체가 값을 반환합니다. 모든 arm의 반환 타입은 동일해야 합니다.

완전성 검사(exhaustiveness check) — 빠뜨린 케이스가 있으면 런타임 오류가 아닌 컴파일 에러가 발생합니다.
📖 The Rust Book §19.1 Rust Reference — match-expr
rs
ex01_match_basic.rs
fn classify(n: i32) -> &'static str {
// match는 표현식 — 반환값을 변수에 바인딩 가능
match n {
i32::MIN..=-1 => "음수",
0 => "영",
1 | 2 | 3 => "작은 양수", // OR 패턴
4..=9 => "한 자리 양수", // 범위 패턴
10..=99 => "두 자리",
_ => "세 자리 이상", // 와일드카드
}
}
fn main() {
for n in [-3, 0, 2, 7, 42, 999] {
println!("{:>4} → {}", n, classify(n));
}
}
RustUTF-8LF
▶ 출력
-3 → 음수 0 → 영 2 → 작은 양수 7 → 한 자리 양수 42 → 두 자리 999 → 세 자리 이상
1-2. Option<T> 매칭
Option<T>는 Some(T) 또는 None인 열거형입니다. Rust에는 null이 없고, 값의 부재를 타입 시스템으로 표현합니다. match로 처리할 때 두 케이스를 모두 처리해야 컴파일됩니다.
⚠️ if let / while let은 exhaustiveness를 검사하지 않습니다. 빠뜨린 케이스가 있어도 컴파일러가 경고하지 않으므로, 모든 케이스 처리가 필요하면 match를 사용하세요.
📖 std::option::Option The Rust Book §6.1
rs
ex02_option.rs
fn find_user(id: u32) -> Option<String> {
match id {
1 => Some("Alice".to_string()),
2 => Some("Bob".to_string()),
_ => None,
}
}
fn main() {
// 1) 완전한 match — 두 케이스 모두 처리
match find_user(1) {
Some(name) => println!("찾음: {}", name),
None => println!("없음"),
}
// 2) if let — 한 패턴만 관심 있을 때 (exhaustiveness 검사 없음)
if let Some(name) = find_user(2) {
println!("if let: {}", name);
}
// 3) while let — 조건이 맞는 동안 반복
let mut stack = vec![10, 20, 30];
while let Some(top) = stack.pop() {
print!("{} ", top);
}
println!();
// 4) 메서드 체인 — match 없이 간결하게
let result = find_user(99)
.map(|s| s.to_uppercase())
.unwrap_or("UNKNOWN".to_string());
println!("결과: {}", result);
}
RustUTF-8LF
▶ 출력
찾음: Alice if let: Bob 30 20 10 결과: UNKNOWN
1-3. Result<T, E> 매칭
Result<T, E>는 Ok(T) 또는 Err(E)인 열거형입니다. 에러 처리의 핵심 타입이며 두 케이스를 모두 처리해야 합니다. ? 연산자는 Err일 때 즉시 호출자에게 전파하는 문법 설탕입니다.
📖 std::result::Result The Rust Book §9.2
rs
ex03_result.rs
fn parse_positive(s: &str) -> Result<u32, String> {
match s.trim().parse::<i32>() {
Ok(n) if n >= 0 => Ok(n as u32),
Ok(_) => Err("음수 불허".to_string()),
Err(e) => Err(format!("파싱 실패: {}", e)),
}
}
// ? 연산자 — Err이면 호출자에게 즉시 전파
fn doubled(s: &str) -> Result<u32, String> {
let n = parse_positive(s)?;
Ok(n * 2)
}
fn main() {
for s in ["42", "-5", "abc", " 7 "] {
match doubled(s) {
Ok(v) => println!("{:>7} → Ok({})", s.trim(), v),
Err(e) => println!("{:>7} → Err({})", s.trim(), e),
}
}
}
RustUTF-8LF
▶ 출력
42 → Ok(84) -5 → Err(음수 불허) abc → Err(파싱 실패: invalid digit found in string) 7 → Ok(14)
1-4. 구조 분해 (Destructuring)
복합 타입(튜플, 구조체, 열거형, 슬라이스)의 내부 값을 패턴으로 분리해 바인딩합니다. ..로 나머지 필드를 한꺼번에 무시할 수 있고, 중첩 구조도 한 번에 분해할 수 있습니다.
📖 The Rust Book §19.3 — Destructuring
rs
ex04_destructure.rs
#[derive(Debug)]
struct Point { x: i32, y: i32 }
#[derive(Debug)]
enum Shape {
Circle { center: Point, radius: f64 },
Line(Point, Point),
}
fn main() {
// 튜플 분해
let (a, b, _) = (1, 2, 3);
println!("a={} b={}", a, b);
// 구조체 분해 — .. 로 나머지 무시
let p = Point { x: 10, y: 20 };
let Point { x, .. } = p;
println!("x={}", x);
// enum 분해
let shapes = vec![
Shape::Circle { center: Point { x: 0, y: 0 }, radius: 5.0 },
Shape::Line(Point { x: 0, y: 0 }, Point { x: 3, y: 4 }),
];
for s in &shapes {
match s {
Shape::Circle { center: Point { x, y }, radius } =>
println!("원: 중심({},{}) r={}", x, y, radius),
Shape::Line(Point { x: x1, y: y1 }, Point { x: x2, y: y2 }) =>
println!("선: ({},{})→({},{})", x1, y1, x2, y2),
}
}
// 슬라이스 분해
let nums = [1, 2, 3, 4, 5];
if let [first, .., last] = nums {
println!("첫={} 마지막={}", first, last);
}
}
RustUTF-8LF
▶ 출력
a=1 b=2 x=10 원: 중심(0,0) r=5 선: (0,0)→(3,4) 첫=1 마지막=5
1-5. 열거형(enum) 매칭
Rust의 enum은 태그 유니언(tagged union)입니다. 각 variant가 서로 다른 데이터를 담을 수 있습니다. 새 variant를 추가하면 match가 있는 모든 곳에서 컴파일 에러가 발생해 누락을 방지합니다.
📖 The Rust Book §6.2
rs
ex05_enum_match.rs
#[derive(Debug)]
enum Command {
Quit, // unit variant
Move { x: i32, y: i32 }, // struct variant
Write(String), // tuple variant
ChangeColor(u8, u8, u8), // tuple variant (RGB)
}
fn execute(cmd: &Command) {
match cmd {
Command::Quit =>
println!("[종료]"),
Command::Move { x, y } =>
println!("[이동] x={} y={}", x, y),
Command::Write(msg) =>
println!("[출력] {}", msg),
Command::ChangeColor(r, g, b) =>
println!("[색상] rgb({},{},{})", r, g, b),
}
}
fn main() {
let cmds = vec![
Command::Move { x: 10, y: -5 },
Command::Write("Hello Rust".to_string()),
Command::ChangeColor(255, 128, 0),
Command::Quit,
];
for cmd in &cmds { execute(cmd); }
}
RustUTF-8LF
▶ 출력
[이동] x=10 y=-5 [출력] Hello Rust [색상] rgb(255,128,0) [종료]
1-6. 가드 (Match Guard)
가드는 패턴 뒤에 붙이는 추가 if 조건입니다. 가드 평가 전에 scrutinee의 해당 부분에 공유 참조가 먼저 생성되고, 가드가 true일 때만 실제로 이동/복사가 일어납니다.
⚠️ 가드가 있는 arm은 exhaustiveness 검사에서 카운트되지 않습니다. Some(x) if x > 0 하나만으로는 완전성이 인정되지 않아 추가 arm이 필요합니다. 또한 가드는 match에서만 사용 가능하며 if let / while let에서는 사용할 수 없습니다.
📖 The Rust Book §19.3 — Match Guards Rust Reference — Match Guards
rs
ex06_guard.rs
fn main() {
// 기본 가드
let num = Some(7);
match num {
Some(x) if x < 5 => println!("5 미만: {}", x),
Some(x) if x < 10 => println!("5~9: {}", x),
Some(x) => println!("10 이상: {}", x),
None => println!("없음"),
}
// 외부 변수 참조 — 가드 안에서 가능
let threshold = 10;
for v in [Some(3), Some(15), None] {
match v {
Some(n) if n < threshold => println!("{} < {}", n, threshold),
Some(n) => println!("{} >= {}", n, threshold),
None => println!("없음"),
}
}
// OR 패턴 + 가드 — 가드는 OR 패턴 전체에 적용
let (x, y) = (5, 0);
match (x, y) {
(4 | 5 | 6, n) if n == 0 => println!("x=4|5|6 이고 y=0"),
(x, _) => println!("기타 x={}", x),
}
}
RustUTF-8LF
▶ 출력
5~9: 7 3 < 10 15 >= 10 없음 x=4|5|6 이고 y=0
1-7. @ 바인딩
@ 연산자는 패턴 검사와 값 캡처를 동시에 수행합니다. n @ 1..=10은 "1~10 범위인지 검사하고, 그 값을 n에 바인딩"합니다. 가드 없이도 조건을 만족하는 값을 변수에 담을 수 있습니다.
📖 The Rust Book §19.3 — @ Bindings
rs
ex07_at_binding.rs
#[derive(Debug)]
enum Message { Hello { id: i32 } }
fn main() {
// @ 바인딩 기본
let x = 7;
match x {
n @ 1..=5 => println!("1~5: {}", n),
n @ 6..=10 => println!("6~10: {}", n),
n => println!("범위 밖: {}", n),
}
// 구조체 필드에 @ 적용
match (Message::Hello { id: 5 }) {
Message::Hello { id: v @ 3..=7 } =>
println!("id {} 는 3~7", v),
Message::Hello { id } =>
println!("기타 id: {}", id),
}
// Option 내부 + outer 동시 캡처
let opt = Some(42);
match opt {
outer @ Some(inner) =>
println!("outer={:?} inner={}", outer, inner),
None => println!("없음"),
}
// 점수 등급 — @ 없이는 값 재사용 불가
let grade = match 87_u32 {
s @ 90..=100 => format!("A ({}점)", s),
s @ 80..=89 => format!("B ({}점)", s),
s @ 70..=79 => format!("C ({}점)", s),
s => format!("F ({}점)", s),
};
println!("등급: {}", grade);
}
RustUTF-8LF
▶ 출력
6~10: 7 id 5 는 3~7 outer=Some(42) inner=42 등급: B (87점)
1-8. matches! 매크로
matches!(expr, pattern)는 패턴이 일치하면 true, 아니면 false를 반환합니다. 가드도 함께 사용할 수 있습니다: matches!(v, Some(x) if x > 0)
📖 std::matches!
rs
ex08_matches.rs
#[derive(Debug)]
enum Status { Active, Inactive, Pending(u32) }
fn main() {
let s = Status::Pending(42);
// 기본 사용
println!("Pending? {}", matches!(s, Status::Pending(_)));
// 가드 포함
println!("n>10? {}", matches!(s, Status::Pending(n) if n > 10));
// 필터링에 활용
let list = vec![
Status::Active, Status::Pending(5),
Status::Inactive, Status::Pending(99), Status::Active,
];
let active = list.iter()
.filter(|s| matches!(s, Status::Active))
.count();
println!("Active 개수: {}", active);
}
RustUTF-8LF
▶ 출력
Pending? true n>10? true Active 개수: 2
Chapter 2 — Entry API
2-1. Entry란?
Entry는 HashMap / BTreeMap에서 특정 키에 대한 자리(slot) 핸들입니다. entry(key)를 호출하면 해시 탐색을 1회만 수행한 뒤, 키가 있으면 Occupied, 없으면 Vacant를 반환합니다.

공식 정의:
pub enum Entry<'a, K, V> {
  Occupied(OccupiedEntry<'a, K, V>),
  Vacant(VacantEntry<'a, K, V>),
}

라이프타임 'a는 Entry가 맵을 빌리는 기간입니다. Entry가 살아있는 동안 맵에 대한 다른 가변 참조를 가질 수 없습니다.
📖 std::collections::hash_map::Entry std::collections::HashMap
rs
ex09_entry_intro.rs
use std::collections::HashMap;
use std::collections::hash_map::Entry;
fn main() {
let mut map: HashMap<&str, u32> = HashMap::new();
// ❌ 이중 탐색 — contains_key + insert = 해시 탐색 2회
if !map.contains_key("score") {
map.insert("score", 0);
}
// ✅ Entry API — 해시 탐색 1회
map.entry("score").or_insert(0);
// entry()가 반환하는 타입 직접 확인
match map.entry("score") {
Entry::Occupied(o) =>
println!("Occupied: 현재값={}", o.get()),
Entry::Vacant(v) =>
println!("Vacant: 키={}", v.key()),
}
match map.entry("new_key") {
Entry::Occupied(o) =>
println!("Occupied: {}", o.get()),
Entry::Vacant(v) =>
println!("Vacant: 키={}", v.key()),
}
}
RustUTF-8LF
▶ 출력
Occupied: 현재값=0 Vacant: 키=new_key
2-2. VacantEntry — 키가 없을 때
• v.key() → &K — 삽입될 키 참조
• v.insert(value) → &mut V — 값 삽입 후 가변 참조 반환 (삽입 직후 바로 수정 가능)
• v.into_key() → K — 삽입 포기하고 키 소유권만 회수
📖 std::collections::hash_map::VacantEntry
rs
ex10_vacant.rs
use std::collections::HashMap;
use std::collections::hash_map::Entry;
fn main() {
let mut map: HashMap<&str, u32> = HashMap::new();
// v.key() — 삽입 전에 키 이름 확인 (공식 문서 예제)
if let Entry::Vacant(v) = map.entry("poneyland") {
println!("삽입할 키: {}", v.key());
v.insert(37);
}
assert_eq!(map["poneyland"], 37);
// v.insert() 반환값 = &mut V — 삽입 직후 즉시 수정
if let Entry::Vacant(v) = map.entry("score") {
let val = v.insert(100);
*val += 50; // 삽입 직후 바로 수정 가능
}
println!("score: {}", map["score"]);
// v.into_key() — 삽입 포기, 키만 회수
let key = match map.entry("temp") {
Entry::Vacant(v) => v.into_key(),
Entry::Occupied(o) => o.key(),
};
println!("회수된 키: {}, temp 존재: {}",
key, map.contains_key("temp"));
}
RustUTF-8LF
▶ 출력
삽입할 키: poneyland score: 150 회수된 키: temp, temp 존재: false
2-3. OccupiedEntry — 키가 있을 때
• o.get() → &V — 불변 참조
• o.get_mut() → &mut V — 가변 참조 (여러 번 호출 가능)
• o.into_mut() → &mut V — Entry를 소비하고 맵 자체의 라이프타임에 바인딩된 가변 참조
• o.insert(v) → V — 값 교체, 이전 값 반환
• o.remove() → V — 제거 후 값 반환
• o.remove_entry() → (K, V) — 키+값 모두 반환
get_mut()은 Entry가 살아있는 동안만 참조가 유효하고, into_mut()은 Entry를 소비하므로 맵의 라이프타임에 바인딩된 더 오래 사는 참조를 반환합니다.
📖 std::collections::hash_map::OccupiedEntry
rs
ex11_occupied.rs
use std::collections::HashMap;
use std::collections::hash_map::Entry;
fn main() {
let mut map = HashMap::from([
("hp", 100_u32), ("mp", 50_u32), ("level", 1_u32),
]);
// o.get_mut() — 여러 번 호출 가능
if let Entry::Occupied(mut o) = map.entry("hp") {
*o.get_mut() -= 30;
*o.get_mut() -= 20;
println!("hp: {}", o.get());
}
// o.insert() — 값 교체, 이전 값 반환
if let Entry::Occupied(mut o) = map.entry("level") {
let old = o.insert(5);
println!("레벨: {} → {}", old, o.get());
}
// o.remove_entry() — 키+값 모두 반환
if let Entry::Occupied(o) = map.entry("mp") {
let (k, v) = o.remove_entry();
println!("제거: 키={} 값={}", k, v);
}
// into_mut() — Entry 소비 후 맵 라이프타임 바인딩
map.insert("gold", 0);
let gold = match map.entry("gold") {
Entry::Occupied(o) => o.into_mut(),
Entry::Vacant(v) => v.insert(0),
};
*gold += 500;
println!("gold: {}", map["gold"]);
}
RustUTF-8LF
▶ 출력
hp: 50 레벨: 1 → 5 제거: 키=mp 값=50 gold: 500
2-4. 공통 편의 메서드
• or_insert(v) — 없으면 v 삽입. 인자는 항상 평가됨
• or_insert_with(f) — 없으면 f() 실행. Vacant일 때만 클로저 실행 (지연 평가)
• or_insert_with_key(f) — 없으면 f(&key) 실행
• or_default() — 없으면 Default::default() 삽입
• and_modify(f) — 있으면 f(&mut V) 실행. 체이닝 가능
⚠️ or_insert(expensive_fn())는 키가 있어도 항상 expensive_fn()을 호출합니다. 비용이 큰 초기화는 반드시 or_insert_with(|| expensive_fn())를 사용하세요.
📖 std::collections::hash_map::Entry — methods
rs
ex12_convenience.rs
use std::collections::HashMap;
fn main() {
let mut map: HashMap<&str, u32> = HashMap::new();
// or_insert — 공식 문서 예제
map.entry("poneyland").or_insert(3);
assert_eq!(map["poneyland"], 3);
*map.entry("poneyland").or_insert(10) *= 2;
assert_eq!(map["poneyland"], 6); // 이미 있으므로 10 무시, 3*2=6
println!("or_insert: {}", map["poneyland"]);
// or_insert_with — Vacant일 때만 클로저 실행
let s = "hoho".to_string();
let mut map2: HashMap<&str, String> = HashMap::new();
map2.entry("poneyland").or_insert_with(|| s.clone());
assert_eq!(map2["poneyland"], "hoho");
println!("or_insert_with: {}", map2["poneyland"]);
// or_default — Option의 Default = None
let mut map3: HashMap<&str, Option<u32>> = HashMap::new();
map3.entry("poneyland").or_default();
assert_eq!(map3["poneyland"], None);
println!("or_default: {:?}", map3["poneyland"]);
// and_modify + or_insert 체이닝 — 공식 문서 예제
let mut map4: HashMap<&str, u32> = HashMap::new();
map4.entry("poneyland").and_modify(|e| *e += 1).or_insert(42);
assert_eq!(map4["poneyland"], 42); // 없었으므로 42
map4.entry("poneyland").and_modify(|e| *e += 1).or_insert(42);
assert_eq!(map4["poneyland"], 43); // 있었으므로 42+1=43
println!("and_modify: {}", map4["poneyland"]);
}
RustUTF-8LF

2-5. 전체 메서드 비교표
대상메서드반환설명
Entryor_insert(v)&mut V없으면 v 삽입 (인자 항상 평가)
Entryor_insert_with(f)&mut V없으면 f() 실행 후 삽입 (지연 평가)
Entryor_insert_with_key(f)&mut V없으면 f(&key) 실행 후 삽입
Entryor_default()&mut V없으면 Default::default() 삽입
Entryand_modify(f)Entry있으면 f(&mut V) 실행, 체이닝 가능
Vacantv.key()&K삽입될 키 참조
Vacantv.insert(val)&mut V값 삽입 후 가변 참조 반환
Vacantv.into_key()K삽입 포기, 키 소유권 반환
Occupiedo.key()&K키 참조
Occupiedo.get()&V값 불변 참조
Occupiedo.get_mut()&mut V값 가변 참조 (여러 번 호출 가능)
Occupiedo.into_mut()&mut VEntry 소비, 맵 라이프타임 바인딩
Occupiedo.insert(v)V값 교체, 이전 값 반환
Occupiedo.remove()V제거, 값 반환
Occupiedo.remove_entry()(K, V)제거, 키+값 모두 반환

3-1. 카운팅
*map.entry(key).or_insert(0) += 1 — or_insert(0)이 반환하는 &mut u32를 역참조해 바로 증가시킵니다.
rs
ex13_counting.rs
use std::collections::HashMap;
fn main() {
let text = "the quick brown fox jumps over the lazy dog the fox";
let mut freq: HashMap<&str, u32> = HashMap::new();
for word in text.split_whitespace() {
*freq.entry(word).or_insert(0) += 1;
}
let mut pairs: Vec<_> = freq.iter().collect();
pairs.sort_by(|a, b| b.1.cmp(a.1).then(a.0.cmp(b.0)));
for (w, c) in &pairs {
println!("{:<8} : {}회", w, c);
}
}

3-2. 그룹핑
map.entry(key).or_default().push(value) — or_default()는 Vec의 경우 빈 Vec을 삽입합니다.
rs
ex14_grouping.rs
use std::collections::HashMap;
fn main() {
let students = vec![
("Alice", 'A', 95_u32), ("Bob", 'B', 82),
("Carol", 'A', 91), ("Dave", 'C', 74),
("Eve", 'B', 88), ("Frank", 'A', 97),
];
let mut by_grade: HashMap<char, Vec<&str>> = HashMap::new();
let mut scores: HashMap<char, Vec<u32>> = HashMap::new();
for &(name, grade, score) in &students {
by_grade.entry(grade).or_default().push(name);
scores.entry(grade).or_default().push(score);
}
let mut grades: Vec<char> = by_grade.keys().copied().collect();
grades.sort();
for g in grades {
let avg = scores[&g].iter().sum::<u32>() as f64
/ scores[&g].len() as f64;
println!("{}반 | 평균 {:.1}점 | {:?}", g, avg, by_grade[&g]);
}
}

3-3. 수정 / 초기화
.and_modify(f).or_insert(v) 체이닝 — 있으면 수정, 없으면 초기화합니다. and_modify는 Rust 1.26부터 안정화된 API입니다.
rs
ex15_modify_or_init.rs
use std::collections::HashMap;
fn main() {
let mut exp: HashMap<&str, u32> = HashMap::new();
// 몬스터 처치 — 첫 처치=100, 이후=2배씩
let kills = [
"고블린", "오크", "고블린", "드래곤",
"오크", "오크", "드래곤", "고블린",
];
for m in kills {
exp.entry(m)
.and_modify(|e| *e *= 2) // 있으면 2배
.or_insert(100); // 없으면 100 시작
}
let mut v: Vec<_> = exp.iter().collect();
v.sort_by(|a, b| b.1.cmp(a.1));
for (m, e) in &v {
println!("{:<8} : {} EXP", m, e);
}
println!("총: {} EXP", v.iter().map(|(_, e)| *e).sum::<u32>());
}

3-4. 지연 초기화
or_insert_with(|| expensive_fn()) — 클로저는 Vacant일 때만 실행됩니다. DB 조회, 파일 IO 등 비용이 큰 작업에 사용합니다.
rs
ex16_lazy_init.rs
use std::collections::HashMap;
fn load_config(svc: &str) -> Vec<String> {
println!(" ⚙ [{}] 설정 로딩...", svc);
vec![format!("host={}.server.com", svc), "port=8080".into()]
}
fn main() {
let mut cache: HashMap<String, Vec<String>> = HashMap::new();
let reqs = ["auth", "payment", "auth", "payment", "auth"];
for svc in reqs {
let cfg = cache
.entry(svc.to_string())
.or_insert_with(|| load_config(svc)); // Vacant일 때만 호출
println!(" {} → {}개 설정", svc, cfg.len());
}
println!("요청 {}회 / 실제 로딩 {}회", reqs.len(), cache.len());
}

3-5. 조건부 삭제
Occupied(o)를 명시적으로 매칭한 뒤 조건 검사 후 o.remove() 또는 o.remove_entry()로 제거합니다.
rs
ex17_cond_remove.rs
use std::collections::HashMap;
use std::collections::hash_map::Entry;
fn main() {
let mut stock: HashMap<&str, u32> = HashMap::from([
("사과", 0), ("바나나", 5), ("포도", 50),
("망고", 0), ("딸기", 20),
]);
let keys: Vec<&str> = stock.keys().copied().collect();
let (mut removed, mut kept) = (vec![], vec![]);
for key in keys {
match stock.entry(key) {
Entry::Occupied(o) if *o.get() == 0 => {
removed.push(key); o.remove();
}
Entry::Occupied(_) => kept.push(key),
Entry::Vacant(_) => {}
}
}
removed.sort(); kept.sort();
println!("❌ 제거: {:?}", removed);
println!("✅ 유지: {:?}", kept);
}

▶ 출력
