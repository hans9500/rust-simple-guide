§ 01

함수형이란? — 명령형과의 차이

함수형 프로그래밍은 프로그램을 상태 변경이 아닌 값의 변환으로 표현하는 방식입니다. "어떻게(How)"보다 "무엇을(What)"에 집중합니다.

Rust
 1fn main() {
 2    // 명령형: 상태를 직접 변경
 3    let nums = vec![1, 2, 3, 4, 5, 6];
 4    let mut result = 0;
 5    for x in &nums {
 6        if x % 2 == 0 {
 7            result += x * 2;
 8        }
 9    }
10    println!("명령형: {}", result); // 24
11
12    // 함수형: 변환 파이프라인
13    let result2: i32 = nums.iter()
14        .filter(|&&x| x % 2 == 0)
15        .map(|&x| x * 2)
16        .sum();
17    println!("함수형: {}", result2); // 24
18}
핵심 차이
명령형: 변수를 만들고 → 루프를 돌고 → 상태를 변경
함수형: 데이터를 파이프라인에 흘려보내며 변환
두 방식 모두 같은 결과를 만들지만, 함수형은 의도를 코드로 직접 표현합니다.
§ 02

불변성 — 원본을 바꾸지 않는다

함수형의 핵심 원칙: 원본 데이터를 변경하지 않고 새로운 값을 만들어낸다. Rust에서 Iterator::map은 항상 새 컬렉션을 반환하며 원본은 그대로입니다.

Rust
 1fn main() {
 2    // 불변 — 원본을 바꾸지 않고 새 값 생성
 3    let v = vec![1, 2, 3, 4, 5];
 4
 5    // map은 원본을 바꾸지 않음
 6    let doubled: Vec<i32> = v.iter()
 7        .map(|&x| x * 2)
 8        .collect();
 9
10    println!("원본:   {:?}", v);       // [1,2,3,4,5]
11    println!("doubled:{:?}", doubled); // [2,4,6,8,10]
12
13    // v는 여전히 유효
14    let sum: i32 = v.iter().sum();
15    println!("sum: {}", sum); // 15
16}
왜 불변성이 중요한가
버그의 주요 원인 중 하나는 예상치 못한 상태 변경입니다.
불변성을 지키면 함수가 어떤 외부 상태도 건드리지 않으므로 코드를 추론하기 쉬워집니다.
§ 03

순수 함수 — 같은 입력, 항상 같은 출력

순수 함수(Pure Function)는 두 가지 조건을 만족합니다: ① 같은 입력에 항상 같은 출력, ② 외부 상태를 읽거나 변경하지 않음(부작용 없음).

Rust
 1// 순수 함수 — 같은 입력 → 항상 같은 출력
 2// 외부 상태에 의존하지 않음
 3fn double(x: i32) -> i32 { x * 2 }
 4fn is_even(x: i32) -> bool { x % 2 == 0 }
 5fn add(a: i32, b: i32) -> i32 { a + b }
 6
 7fn main() {
 8    println!("{}", double(5));   // 항상 10
 9    println!("{}", double(5));   // 항상 10
10    println!("{}", is_even(4));  // 항상 true
11    println!("{}", add(3, 4));   // 항상 7
12
13    let nums = vec![1, 2, 3, 4, 5];
14    let evens: Vec<i32> = nums.iter()
15        .copied()
16        .filter(|&x| is_even(x)) // 순수 함수 전달
17        .collect();
18    println!("{:?}", evens); // [2, 4]
19}
순수 함수 vs 불순 함수
fn double(x: i32) → i32 → 순수: 외부 의존 없음
fn get_random() → i32 → 불순: 호출마다 다른 값
fn print_and_return(x: i32) → i32 → 불순: 출력(부작용) 발생
순수 함수는 테스트하기 쉽고, 조합하기 쉽고, 추론하기 쉽습니다.
§ 04

고차 함수 — 함수를 값처럼 다루기

고차 함수(Higher-Order Function)는 함수를 인자로 받거나 함수를 반환하는 함수입니다. 이것이 함수형 프로그래밍의 핵심 도구입니다.

Rust
 1// 고차 함수 — 함수를 인자로 받거나 반환
 2fn apply<F: Fn(i32) -> i32>(f: F, x: i32) -> i32 {
 3    f(x)
 4}
 5fn make_adder(n: i32) -> impl Fn(i32) -> i32 {
 6    move |x| x + n // 함수를 반환
 7}
 8fn apply_all<F: Fn(i32) -> i32>(
 9    nums: &[i32], f: F
10) -> Vec<i32> {
11    nums.iter().map(|&x| f(x)).collect()
12}
13fn main() {
14    println!("{}", apply(|x| x * 3, 5)); // 15
15    let add10 = make_adder(10);
16    println!("{}", add10(5));  // 15
17    println!("{}", add10(20)); // 30
18
19    let nums = vec![1, 2, 3];
20    println!("{:?}", apply_all(&nums, |x| x * x));
21    // [1, 4, 9]
22}
함수형의 힘
make_adder(10)이 반환하는 것은 값이 아니라 행동(클로저)입니다.
함수를 값처럼 저장하고 전달할 수 있으면 코드 재사용성이 극적으로 높아집니다.
§ 05

map · filter · fold — 3가지 핵심 도구

함수형 프로그래밍의 세 기둥: map(변환), filter(선별), fold(누산). 이 세 가지로 대부분의 데이터 처리를 표현할 수 있습니다.

Rust
 1fn main() {
 2    let nums = vec![1, 2, 3, 4, 5];
 3
 4    // map — 각 원소를 변환
 5    let sq: Vec<i32> = nums.iter()
 6        .map(|&x| x * x)
 7        .collect();
 8    println!("map:    {:?}", sq); // [1,4,9,16,25]
 9
10    // filter — 조건을 만족하는 것만
11    let big: Vec<&i32> = nums.iter()
12        .filter(|&&x| x > 3)
13        .collect();
14    println!("filter: {:?}", big); // [4, 5]
15
16    // fold — 누산 (초기값 + 클로저)
17    let sum = nums.iter()
18        .fold(0, |acc, &x| acc + x);
19    println!("fold:   {}", sum); // 15
20
21    // 체이닝
22    let result: i32 = nums.iter()
23        .filter(|&&x| x % 2 != 0)
24        .map(|&x| x * 10)
25        .fold(0, |acc, x| acc + x);
26    println!("chain:  {}", result); // 90
27}
fold가 가장 강력한 이유
sum(), product(), max(), count() 등 모든 집계 연산은 사실 fold의 특수한 경우입니다.
fold(초기값, |누산값, 원소| → 새누산값)
§ 06

Lazy 평가 — 필요할 때만 실행

Rust Iterator의 가장 중요한 특성: 어댑터는 실행을 미루고, 소비자가 호출될 때 비로소 실행됩니다. 이것을 Lazy 평가라고 합니다.

Rust
 1fn main() {
 2    // Lazy — 소비자 없으면 실행 안 됨
 3    let _lazy = vec![1, 2, 3].iter()
 4        .map(|x| { println!("map 실행됨!"); x * 2 });
 5    // 위 코드: println! 한 번도 출력 안 됨
 6    println!("소비자 없음 — 위 map 실행 안 됨");
 7
 8    // collect()가 소비자 → 그때 map 실행
 9    let result: Vec<i32> = vec![1, 2, 3].iter()
10        .map(|x| { print!("[{}->{}] ", x, x*2); x * 2 })
11        .collect();
12    println!("
13결과: {:?}", result);
14    // [1->2] [2->4] [3->6]
15    // 결과: [2, 4, 6]
16}
vec![1,2,3,4,5]
.iter()소스
.filter()x%2==0
.map()x*10
.take(2)최대 2개
.collect()실행 트리거
원본 데이터 — 클릭해서 값 변경
collect()가 take.next()를 호출하면 시작됩니다
처리0
통과0
결과[]
실행 로그 — ← pull / → 데이터 흐름
Lazy 평가 규칙
어댑터: .map() .filter() .take() → Iterator 반환, 실행 없음
소비자: .collect() .sum() .for_each() → 값 반환, 이때 실행
pull 방향: 소비자 → take → map → filter → iter → 데이터
§ 07

take · skip · zip · flat_map — 어댑터 심화

Lazy 평가 덕분에 어댑터를 자유롭게 조합할 수 있습니다. 무한 Iterator도 take와 함께라면 안전하게 사용 가능합니다.

Rust
 1fn main() {
 2    // take — n개에서 중단 (단락 평가)
 3    // 무한 Iterator도 가능!
 4    let first5: Vec<i32> = (0..)   // 0,1,2,3,...무한
 5        .filter(|x| x % 2 == 0)    // 짝수만
 6        .take(5)                    // 5개에서 중단
 7        .collect();
 8    println!("{:?}", first5); // [0,2,4,6,8]
 9
10    // take_while — 조건 만족까지
11    let v = vec![1, 3, 5, 2, 4, 6];
12    let odd_prefix: Vec<&i32> = v.iter()
13        .take_while(|&&x| x % 2 != 0)
14        .collect();
15    println!("{:?}", odd_prefix); // [1, 3, 5]
16
17    // skip — 앞 n개 건너뜀
18    let after3: Vec<&i32> = v.iter()
19        .skip(3)
20        .collect();
21    println!("{:?}", after3); // [2, 4, 6]
22}
Rust
 1fn main() {
 2    let a = vec![1, 2, 3];
 3    let b = vec![4, 5, 6];
 4
 5    // zip — 두 Iterator를 쌍으로
 6    let pairs: Vec<_> = a.iter()
 7        .zip(b.iter())
 8        .collect();
 9    println!("zip: {:?}", pairs);
10    // [(1,4),(2,5),(3,6)]
11
12    // enumerate — 인덱스 + 값
13    for (i, v) in a.iter().enumerate() {
14        print!("[{}]={} ", i, v);
15    }
16    println!();
17
18    // flat_map — 중첩 Iterator 평탄화
19    let words = vec!["hello world", "foo bar"];
20    let chars: Vec<&str> = words.iter()
21        .flat_map(|s| s.split_whitespace())
22        .collect();
23    println!("{:?}", chars);
24    // ["hello","world","foo","bar"]
25}
Rust
 1fn main() {
 2    // chain — 두 Iterator 연결
 3    let a = vec![1, 2, 3];
 4    let b = vec![10, 20, 30];
 5    let chained: Vec<&i32> = a.iter()
 6        .chain(b.iter())
 7        .collect();
 8    println!("chain: {:?}", chained);
 9    // [1,2,3,10,20,30]
10
11    // peekable — 다음 원소 미리 보기
12    let v = vec![1, 2, 3, 4, 5];
13    let mut it = v.iter().peekable();
14    while let Some(&x) = it.next() {
15        if let Some(&&next) = it.peek() {
16            print!("{}→{} ", x, next);
17        } else {
18            print!("{}(끝)", x);
19        }
20    }
21    println!();
22    // 1→2 2→3 3→4 4→5 5(끝)
23}
💡 어댑터 선택 기준
take / take_while: 개수/조건으로 앞에서 자름 — 무한 Iterator에 필수
skip / skip_while: 앞부분 건너뜀
zip: 두 Iterator를 쌍으로 — 길이가 다르면 짧은 쪽에서 종료
flat_map: 각 원소가 Iterator를 반환할 때 평탄화
chain: 두 Iterator를 순서대로 연결
§ 08

함수 합성 — 작은 함수를 조합하기

함수 합성(Function Composition)은 작고 순수한 함수 여러 개를 연결해 복잡한 기능을 만드는 패턴입니다. g(f(x)) 형태로, 각 함수는 하나의 일만 합니다.

Rust
 1// 함수 합성 — 작은 함수를 조합해 큰 기능
 2fn compose<A,B,C,F,G>(f: F, g: G)
 3    -> impl Fn(A) -> C
 4where F: Fn(A)->B, G: Fn(B)->C {
 5    move |x| g(f(x))
 6}
 7fn main() {
 8    let double   = |x: i32| x * 2;
 9    let add_one  = |x: i32| x + 1;
10    let to_str   = |x: i32| x.to_string();
11
12    // double 후 add_one
13    let double_then_add = compose(double, add_one);
14    println!("{}", double_then_add(5)); // 11
15
16    // 파이프라인: 3 → 6 → 7 → "7"
17    let pipeline = compose(
18        compose(double, add_one), to_str
19    );
20    println!("{}", pipeline(3)); // "7"
21
22    // Iterator로 파이프라인
23    let ops: Vec<Box<dyn Fn(i32)->i32>> = vec![
24        Box::new(|x| x * 2),
25        Box::new(|x| x + 1),
26        Box::new(|x| x * x),
27    ];
28    let result = ops.iter().fold(3, |v, f| f(v));
29    println!("{}", result); // (3*2+1)^2 = 49
30}
파이프라인 패턴
Iterator 체인 자체가 함수 합성입니다.
nums.iter().filter(f).map(g).fold(h) = h(g(f(nums)))
§ 09

클로저로 상태 캡슐화

클로저는 주변 환경을 캡처합니다. 이를 이용하면 상태를 가진 함수를 만들 수 있습니다. 외부에 상태를 노출하지 않아 안전합니다.

Rust
 1// 클로저로 상태 캡슐화
 2fn make_counter(start: i32, step: i32)
 3    -> impl FnMut() -> i32 {
 4    let mut n = start;
 5    move || { let v = n; n += step; v }
 6}
 7fn make_running_avg()
 8    -> impl FnMut(f64) -> f64 {
 9    let mut sum = 0.0;
10    let mut cnt = 0;
11    move |x| {
12        sum += x; cnt += 1;
13        sum / cnt as f64
14    }
15}
16fn main() {
17    let mut by2 = make_counter(0, 2);
18    println!("{} {} {}", by2(), by2(), by2());
19    // 0 2 4
20
21    let mut avg = make_running_avg();
22    for x in [10.0, 20.0, 30.0, 40.0] {
23        println!("avg: {:.1}", avg(x));
24    }
25    // 10.0  15.0  20.0  25.0
26}
클로저 캡슐화의 장점
make_counter()가 반환하는 클로저는 내부 상태(n)를 외부에 노출하지 않습니다.
각 인스턴스는 독립적인 상태를 가집니다 — by2와 by3은 서로 영향 없음
§ 10

Iterator 직접 구현 — impl Iterator

Iterator 트레이트를 구현하면 자신만의 Iterator를 만들 수 있습니다. 단 하나의 메서드 next()만 구현하면 map, filter, fold 등 수십 개의 메서드를 공짜로 얻습니다.

Rust
 1// Iterator 직접 구현
 2struct Range { cur: i32, end: i32, step: i32 }
 3impl Range {
 4    fn new(start: i32, end: i32, step: i32)
 5        -> Self {
 6        Range { cur: start, end, step }
 7    }
 8}
 9impl Iterator for Range {
10    type Item = i32;
11    fn next(&mut self) -> Option<i32> {
12        if self.cur >= self.end { return None; }
13        let v = self.cur;
14        self.cur += self.step;
15        Some(v)
16    }
17}
18fn main() {
19    let r = Range::new(0, 10, 2);
20    let v: Vec<i32> = r.collect();
21    println!("{:?}", v); // [0,2,4,6,8]
22
23    // 표준 어댑터 전부 사용 가능!
24    let sum: i32 = Range::new(1, 6, 1).sum();
25    println!("sum: {}", sum); // 15
26
27    let big: Vec<i32> = Range::new(0, 20, 3)
28        .filter(|&x| x > 10)
29        .collect();
30    println!("{:?}", big); // [12, 15, 18]
31}
Iterator 트레이트의 힘
next() 하나만 구현하면 표준 라이브러리의 모든 어댑터/소비자를 그대로 사용할 수 있습니다.
이것이 트레이트 기반 추상화의 핵심입니다.
§ 11

any · all · find · scan · unzip — 소비자 심화

단순 collect()와 sum() 외에도 다양한 소비자가 있습니다. 모두 Lazy 평가를 따르며, 필요한 순간에만 실행됩니다.

Rust
 1fn main() {
 2    // any / all — 단락 평가
 3    let nums = vec![1, 2, 3, 4, 5];
 4    println!("{}", nums.iter().any(|&x| x > 4));
 5    // true
 6    println!("{}", nums.iter().all(|&x| x > 0));
 7    // true
 8
 9    // find — 첫 번째 일치
10    let first_even = nums.iter()
11        .find(|&&x| x % 2 == 0);
12    println!("{:?}", first_even); // Some(2)
13
14    // position — 인덱스 반환
15    let pos = nums.iter()
16        .position(|&x| x == 3);
17    println!("{:?}", pos); // Some(2)
18
19    // min / max / count
20    println!("min={:?} max={:?} cnt={}",
21        nums.iter().min(),
22        nums.iter().max(),
23        nums.iter().filter(|&&x| x%2==0).count()
24    ); // min=1 max=5 cnt=2
25}
Rust
 1fn main() {
 2    // scan — fold처럼 상태를 유지하며 중간값 반환
 3    let nums = vec![1, 2, 3, 4, 5];
 4    let running_sum: Vec<i32> = nums.iter()
 5        .scan(0, |state, &x| {
 6            *state += x;
 7            Some(*state)
 8        })
 9        .collect();
10    println!("{:?}", running_sum);
11    // [1, 3, 6, 10, 15]
12
13    // unzip — (A,B) 쌍을 두 Vec으로 분리
14    let pairs = vec![(1,'a'),(2,'b'),(3,'c')];
15    let (nums2, chars): (Vec<i32>, Vec<char>)
16        = pairs.into_iter().unzip();
17    println!("{:?}", nums2);  // [1, 2, 3]
18    println!("{:?}", chars);  // ['a','b','c']
19}
💡 소비자 선택 기준
any / all: 단락 평가 — 조건 만족/불만족 즉시 종료
find / position: 첫 번째 일치 — 이후 원소 처리 안 함
scan: fold와 달리 중간값을 Iterator로 반환
min / max / count: 전체 소비 필요 — 무한 Iterator에 사용 불가
§ 12

실전 패턴 — 로그 파싱 · 빈도 집계 · 윈도우

실무에서 함수형 파이프라인이 빛나는 세 가지 패턴: 데이터 파싱, 집계, 슬라이딩 윈도우.

Rust
 1// 실전: 로그 파싱 파이프라인
 2#[derive(Debug)]
 3struct Log { level: String, msg: String }
 4
 5fn parse_log(line: &str) -> Option<Log> {
 6    let parts: Vec<&str> =
 7        line.splitn(2, ':').collect();
 8    if parts.len() == 2 {
 9        Some(Log {
10            level: parts[0].trim().to_string(),
11            msg: parts[1].trim().to_string(),
12        })
13    } else { None }
14}
15
16fn main() {
17    let lines = vec![
18        "ERROR: 디스크 꽉참",
19        "INFO: 서버 시작",
20        "WARN: 메모리 80%",
21        "잘못된 형식",
22        "ERROR: 연결 실패",
23    ];
24    let errors: Vec<String> = lines.iter()
25        .filter_map(|l| parse_log(l))
26        .filter(|l| l.level == "ERROR")
27        .map(|l| l.msg)
28        .collect();
29    println!("{:?}", errors);
30    // ["디스크 꽉참", "연결 실패"]
31}
Rust
 1use std::collections::HashMap;
 2
 3fn main() {
 4    let words = vec![
 5        "apple","banana","apple",
 6        "cherry","banana","apple",
 7    ];
 8
 9    // 빈도 집계 — fold로 HashMap 구성
10    let freq = words.iter().fold(
11        HashMap::new(),
12        |mut map, &w| {
13            *map.entry(w).or_insert(0) += 1;
14            map
15        }
16    );
17
18    // 정렬 후 출력
19    let mut pairs: Vec<_> =
20        freq.iter().collect();
21    pairs.sort_by(|a, b| b.1.cmp(a.1));
22    for (word, cnt) in pairs {
23        println!("{}: {}", word, cnt);
24    }
25    // apple: 3
26    // banana: 2
27    // cherry: 1
28}
Rust
 1fn main() {
 2    // windows — 슬라이딩 윈도우
 3    let data = vec![1.0,3.0,5.0,2.0,8.0,4.0];
 4    let moving_avg: Vec<f64> = data
 5        .windows(3)
 6        .map(|w| {
 7            let s: f64 = w.iter().sum();
 8            (s / w.len() as f64 * 10.0).round()
 9                / 10.0
10        })
11        .collect();
12    println!("{:?}", moving_avg);
13    // [3.0, 3.3, 5.0, 4.7]
14
15    // chunks — 고정 크기로 분할
16    let batch: Vec<Vec<&f64>> = data
17        .chunks(2)
18        .map(|c| c.iter().collect())
19        .collect();
20    for b in &batch {
21        println!("{:?}", b);
22    }
23}
💡 실전 파이프라인 패턴
filter_map: None을 자동 제거하며 변환 — parse, try_into 등에 유용
fold로 HashMap: entry API와 조합해 빈도 집계
windows / chunks: 슬라이딩 윈도우 / 고정 크기 분할 — 시계열 데이터에 적합
§ 13

Option · Result도 함수형으로

Option과 Result도 Iterator처럼 map, filter, and_then을 지원합니다. 에러 처리도 함수형 파이프라인으로 표현할 수 있습니다.

Rust
 1// Option/Result 도 함수형으로
 2fn parse_port(s: &str) -> Option<u16> {
 3    s.parse().ok().filter(|&p| p > 0)
 4}
 5
 6fn main() {
 7    // Option 체이닝
 8    let port = parse_port("8080")
 9        .map(|p| p + 1)
10        .unwrap_or(80);
11    println!("port: {}", port); // 8081
12
13    // Result 체이닝
14    let results: Vec<Result<i32, _>> = vec![
15        "1","two","3","four","5"
16    ].iter()
17     .map(|s| s.parse::<i32>())
18     .collect();
19
20    // 성공만 필터링
21    let ok: Vec<i32> = results.iter()
22        .filter_map(|r| r.as_ref().ok().copied())
23        .collect();
24    println!("{:?}", ok); // [1, 3, 5]
25
26    // 실패 개수
27    let err_cnt = results.iter()
28        .filter(|r| r.is_err()).count();
29    println!("실패: {}", err_cnt); // 2
30}
filter_map의 역할
filter_map은 filter + map을 한 번에 처리합니다.
Option을 반환하는 함수를 전달하면 None을 자동으로 걸러냅니다.
§ 14

iter · iter_mut · into_iter — 소유권과 Iterator

Rust에서 Iterator를 만드는 세 가지 방법은 소유권 규칙과 직결됩니다. 올바른 선택이 중요합니다.

Rust
 1fn main() {
 2    // iter()      — 불변 참조 &T
 3    // iter_mut()  — 가변 참조 &mut T
 4    // into_iter() — 소유권 이전 T
 5
 6    let v = vec![1, 2, 3];
 7
 8    // iter() — 원본 유지
 9    let sum: i32 = v.iter().sum();
10    println!("sum={}, v 여전히 유효: {:?}", sum, v);
11
12    // iter_mut() — 원소 수정
13    let mut v2 = vec![1, 2, 3];
14    v2.iter_mut().for_each(|x| *x *= 10);
15    println!("{:?}", v2); // [10,20,30]
16
17    // into_iter() — 소유권 이전
18    let v3 = vec![
19        "hello".to_string(),
20        "world".to_string(),
21    ];
22    let upper: Vec<String> = v3.into_iter()
23        .map(|s| s.to_uppercase())
24        .collect();
25    println!("{:?}", upper);
26    // ["HELLO", "WORLD"]
27    // println!("{:?}", v3); // ❌ 이동됨
28}
선택 기준
원본 유지 필요 → iter() &T
원소 직접 수정 → iter_mut() &mut T
소유권 이전 + 변환 → into_iter() T
§ 15

명령형 vs 함수형 — 패턴 비교

같은 문제를 두 방식으로 표현했을 때 가독성과 의도의 명확성을 비교합니다.

Rust
 1fn main() {
 2    // 같은 문제 — 두 가지 방식 비교
 3    let data = vec![
 4        ("Alice", 85), ("Bob", 42),
 5        ("Carol", 91), ("Dave", 38),
 6        ("Eve", 77),
 7    ];
 8
 9    // 명령형
10    let mut passing_imp = Vec::new();
11    for (name, score) in &data {
12        if *score >= 60 {
13            passing_imp.push(
14                format!("{}: {}", name, score)
15            );
16        }
17    }
18    println!("명령형: {:?}", passing_imp);
19
20    // 함수형
21    let passing_func: Vec<String> = data.iter()
22        .filter(|(_, s)| *s >= 60)
23        .map(|(n, s)| format!("{}: {}", n, s))
24        .collect();
25    println!("함수형: {:?}", passing_func);
26
27    // 결과 동일, 코드 의도가 더 명확
28}
언제 어느 방식을?
함수형: 데이터 변환/집계/필터링 → 파이프라인이 의도를 명확히 표현
명령형: 복잡한 제어 흐름, 조기 종료가 많은 경우 → 직접 루프가 더 명확
Rust는 두 방식 모두 지원합니다 — 상황에 맞게 선택하세요.
