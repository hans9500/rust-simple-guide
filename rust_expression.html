공식 문서의 근본 정의

"Rust is primarily an expression language." — Rust Reference § 6 Statements and expressions

이 한 문장이 Rust 문법의 핵심입니다. Java, C 와 달리 Rust 에서는 거의 모든 구문이 expression 입니다. if, match, loop, 심지어 블록 { } 까지 모두 값을 만들어냅니다. Statement 는 그 중에서 "값을 목적으로 하지 않는 특별한 형태"일 뿐입니다.

그림 2 — Rust 코드 단위 분류 트리
Java / C 와의 결정적 차이

Java 에서 if 는 statement 입니다. 값을 반환하지 않습니다 (삼항 연산자 ?: 는 expression). Rust 에서 if 는 expression 입니다. 값을 반환합니다. 이 차이가 Rust 코드 스타일 전체를 결정합니다.

공식 문서 용어 정리

Expression (표현식)
평가(evaluate)되어 값(value)을 만들어내는 코드 조각. 반드시 어떤 타입을 가집니다.
Statement (문장)
블록 안에서 실행되는 단위. 값을 "생산"하지 않습니다. 공식 문서 기준 세 종류만 존재합니다.
Block expression (블록 표현식)
{ } 로 감싼 statement 와 expression 의 시퀀스. 그 자체가 expression 입니다.
Tail expression (꼬리 표현식)
블록의 맨 마지막에 세미콜론 없이 오는 expression. 블록 전체의 값이 됩니다.
Scrutinee (심사 대상)
match 에 넘기는 값. 공식 문서가 사용하는 정식 용어입니다.
Irrefutable pattern (반박 불가 패턴)
항상 매칭되는 패턴. let 에서는 이것만 사용 가능합니다.
2

Expression — 정확한 정의와 예제

공식 문서 (Reference § Expressions): "An expression may have two roles: it always produces a value, and it may have effects (otherwise known as 'side effects')."

Expression 은 두 가지 일을 동시에 합니다. 항상 어떤 값을 만들고, 부수적으로 부작용(화면 출력, 메모리 변경 등)도 가질 수 있습니다.

Place expression vs Value expression

공식 문서 (Reference § Expressions § Place expressions and value expressions) 에서는 expression 을 두 종류로 구분합니다.

Place expression
메모리 위치를 가리키는 expression. 다른 언어의 lvalue 에 해당합니다. 변수(x), 역참조(*p), 필드 접근(s.field), 인덱스(a[0]) 등이 해당합니다. 대입의 왼쪽에 올 수 있습니다.
Value expression
값을 만들어내는 expression. 다른 언어의 rvalue 에 해당합니다. 리터럴, 연산 결과, 함수 호출 결과 등이 해당합니다.
rust — 기본 expression 총람 
fn main() {
    // ─ 리터럴 expression ────────────────────────────────
    let _a: i32  = 42;        // 정수 리터럴
    let _b: f64  = 3.14;      // 부동소수점 리터럴
    let _c: bool = true;       // bool 리터럴
    let _d: &str = "hello";    // 문자열 슬라이스 리터럴
    let _e: char = 'A';        // char 리터럴
    let _f: ()   = ();           // unit 리터럴 — 타입도 () 이고 값도 ()

    // ─ 산술 · 비교 · 논리 연산자 expression ────────────
    let sum   = 3 + 4;           // i32: 7
    let gt    = 10 > 5;          // bool: true
    let both  = true && false;  // bool: false
    let neg   = -42;            // i32: -42  (단항 연산자)
    let not   = !true;          // bool: false

    // ─ 함수 호출 / 메서드 호출도 expression ─────────────
    let s  = String::from("rust"); // String 타입 값
    let n  = s.len();               // usize 타입 값
    let up = s.to_uppercase();      // String 타입 값

    // ─ 구조체 생성도 expression ──────────────────────────
    struct Point { x: i32, y: i32 }
    let p = Point { x: 1, y: 2 };   // Point 타입 값

    // ─ 클로저도 expression ───────────────────────────────
    let add = |a: i32, b: i32| a + b;  // 클로저 타입 값
    let r = add(3, 4);                    // i32: 7

    // ─ 범위(Range)도 expression ──────────────────────────
    let r1 = 1..5;      // Range<i32>
    let r2 = 1..=5;    // RangeInclusive<i32>
}
rust — if / match / loop 모두 expression 
fn main() {
    let x = 7;

    // ─ if 는 expression ──────────────────────────────────
    let label = if x > 0 { "양수" } else { "비양수" };
    // label: &str = "양수"

    // ─ match 는 expression ───────────────────────────────
    let kind = match x % 2 {
        0 => "짝수",
        _ => "홀수",
    };
    // kind: &str = "홀수"

    // ─ loop 도 expression (break 로 값을 반환) ────────────
    let found = loop {
        if x > 5 { break "큰 수"; }  // break 가 값을 들고 나옴
        break "작은 수";
    };
    // found: &str = "큰 수"

    // ─ 블록 { } 도 expression ───────────────────────────
    let computed = {
        let a = x * 2;
        let b = a + 1;
        b       // tail expression — 블록의 값 = 15
    };
    // computed: i32 = 15

    println!("{} / {} / {} / {}", label, kind, found, computed);
    // 양수 / 홀수 / 큰 수 / 15
}
3

Statement — 세 가지 종류

공식 문서 (Reference § Statements): "A statement is a component of a block." Statement 는 오직 블록 안에서만 존재합니다. 공식 문서 기준 정확히 세 종류입니다.

그림 — Statement 세 종류 (분류 트리 참고)

① let statement

rust — let statement 완전 예제 
fn main() {
    // 공식 문서 문법: let PatternNoTopAlt [: Type] [= Expression] ;

    // ─ 기본 ────────────────────────────────────────────
    let x = 5;                  // 타입 추론: i32
    let y: f64 = 3.14;          // 타입 명시
    let z: i32;                  // 초기화 없음 — 사용 전에 반드시 할당
    z = 10;                      // 이후 초기화 OK

    // ─ irrefutable pattern 바인딩 ────────────────────
    // let 은 반드시 irrefutable(항상 매칭) 패턴만 허용
    let (a, b) = (1, 2);         // 튜플 분해
    let [first, second] = [10, 20]; // 배열 분해
    struct Point { x: i32, y: i32 }
    let Point { x: px, y: py } = Point { x: 3, y: 7 };
    // px = 3, py = 7

    // ─ let mut ───────────────────────────────────────
    let mut count = 0;
    count += 1;   // count = 1

    // ─ shadowing (재선언) ─────────────────────────────
    let val = 5;
    let val = val * 2;  // 새 변수 val = 10 (이전 val 은 가려짐)
    let val = "ten";    // 타입도 바뀔 수 있음! mut 으로는 불가능

    // ─ let 은 값을 반환하지 않는다 ───────────────────
    // let w = (let v = 5);  // error[E0658]: let chains are unstable
    println!("{} {} {} {}", a, b, first, px);
}

② item declaration statement

함수(fn), 구조체(struct), 열거형(enum), 타입 별칭(type), use 선언 등이 해당합니다. 이것들은 블록 안에서도 선언 가능합니다. 값이 없으므로 statement 입니다.

rust — item declaration in block 
fn main() {
    // 함수 안에서 fn, struct, enum 선언 가능
    fn double(n: i32) -> i32 { n * 2 }

    struct Pair(i32, i32);

    enum Dir { North, South, East, West }

    let r = double(5);          // 10
    let p = Pair(1, 2);
    let d = Dir::North;

    // item declaration 은 값이 없다 — 다음은 에러
    // let x = fn double(...) { ... };  // error
}

③ expression statement

공식 문서: "An expression statement is one that evaluates an expression and ignores its value." Expression 뒤에 ; 를 붙이면 expression statement 가 됩니다. 값을 평가하지만 결과를 버립니다.

rust — expression statement 예제 
fn main() {
    // 이것들은 모두 expression statement
    // 값을 만들지만 ; 가 그 값을 버립니다

    1 + 1;                    // i32 값 2 생성 후 버림
    "hello".to_uppercase();  // String 생성 후 즉시 drop
    Vec::<i32>::new();       // Vec 생성 후 즉시 drop
    println!("hi");           // () 반환 — 부작용(출력)만 남음

    // match 도 expression — ; 붙이면 expression statement
    let i = 1;
    match i {
        1 => 100,
        _ => 200,
    };   // i32 값 100 이 만들어졌다가 버려짐

    // if expression 도 expression statement 가 될 수 있음
    if i > 0 { println!("pos"); };  // () 를 만들고 버림

    // 블록도 expression statement 가 될 수 있음
    { let tmp = 42; tmp };  // i32 값 42 를 만들고 버림
}
unused value 경고

expression statement 로 값을 버릴 때 컴파일러가 warning: unused value that must be used 경고를 낼 수 있습니다. Result, Option 등 #[must_use] 가 붙은 타입은 반드시 처리해야 합니다. 명시적으로 버리려면 let _ = expr; 또는 drop(expr); 을 사용하세요.

4

세미콜론(;)의 두 가지 역할

세미콜론은 위치에 따라 완전히 다른 역할을 합니다. 이것을 혼동하는 것이 Rust 입문에서 가장 흔한 실수입니다.

그림 3 — 세미콜론의 두 가지 역할

역할 A — expression statement 종결자 (값을 버림)

공식 문서 (Reference § Expression statements): expression 뒤의 ; 는 그 expression 의 값을 버리고 () 타입으로 만듭니다.

rust — 역할 A 상세 
fn main() {
    // expr;  →  expression statement  →  () 타입

    42;               // i32 값 42 를 만들었다가 버림 → ()
    "hello";          // &str 값을 만들었다가 버림 → ()

    let i = 3;
    match i { 1 => 10, _ => 99 };
    //                               ↑ i32 값 99 가 버려짐 → ()

    if i > 0 { "pos" } else { "neg" };
    //                                        ↑ &str 값 "pos" 가 버려짐 → ()
}

역할 B — let statement 의 문법적 종결자 (값을 버리지 않음)

let x = expr; 에서의 ; 는 let 선언 문법의 필수 요소입니다. expression 의 값을 버리는 것이 아니라 let 선언 전체를 끝내는 구두점입니다. expression 의 값은 x 에 바인딩됩니다.

rust — 역할 B 상세 
fn main() {
    // let x = expr;  →  let statement  →  값은 x 에 바인딩
    //                ↑ 이 ; 는 let 문법 종결자 — 값을 버리지 않음!

    let i = 3;
    let result = match i { 1 => 10, _ => 99 };
    //            ↑ match 는 여전히 i32 expression
    //                                              ↑ let 종결자
    // result: i32 = 99

    let flag = if i > 0 { true } else { false };
    // flag: bool = true

    let computed = {
        let a = i * 10;
        a + 5           // tail expression → 블록의 값 35
    };                  // let 종결자
    // computed: i32 = 35

    println!("{} {} {}", result, flag, computed);
}

for / while 루프에서 match 사용 — 자주 하는 실수

rust — for body 는 반드시 () 타입 
fn main() {
    // for body 는 () 타입만 허용

    // ✗ 에러: match 가 i32 를 반환하는데 사용하지 않음
    // for i in 1..5 {
    //     match i { 1 => 10, _ => 20 }  ← i32 반환
    // }  error[E0308]: mismatched types, expected `()`, found `i32`

    // ✓ 해결 1: ; 로 값을 버림
    for i in 1..5 {
        match i { 1 => 10, _ => 20 };   // ; 로 i32 버림 → ()
    }

    // ✓ 해결 2: arm 이 () 를 반환하도록
    for i in 1..5 {
        match i {
            1 => println!("one"),   // println! → ()
            _ => println!("other"),
        }   // match 전체 → () → for body OK
    }

    // ✓ 해결 3: 값을 실제로 사용
    for i in 1..5 {
        let v = match i { 1 => 10, _ => 20 };
        println!("{}", v);
    }
}

5

Block expression — tail expression 과 블록의 값

공식 문서 (Reference § Block expressions): "The value of a block expression is the value of the final operand, or () if the final operand is a statement."

블록의 맨 마지막 expression 을 공식 문서는 tail expression (꼬리 표현식) 이라 부릅니다. 이 tail expression 의 값이 블록 전체의 값이 됩니다.

그림 5 — Block expression 값 결정 규칙

블록 값 결정 규칙 — 4가지 경우

rust — 블록 값 결정 완전 예제 
fn main() {
    // ─ 케이스 1: tail expression 있음 → 그 값이 블록의 값 ─
    let a = {
        let x = 3;    // statement
        let y = 4;    // statement
        x + y          // tail expression (세미콜론 없음!) → 값 7
    };               // a: i32 = 7

    // ─ 케이스 2: 마지막에 ; 있음 → 블록 값 = () ──────────
    let b = {
        let x = 3;
        let y = 4;
        x + y;         // expression statement — 값 버려짐
                        // 블록 마지막이 statement → 블록 값 = ()
    };               // b: () = ()

    // ─ 케이스 3: 빈 블록 → () ────────────────────────────
    let c = {};      // c: () = ()

    // ─ 케이스 4: 부작용 + tail expression ────────────────
    let d = {
        println!("부작용 발생"); // expression statement (부작용)
        42                       // tail expression → 값 42
    };               // d: i32 = 42

    // ─ 케이스 5: 중첩 블록 ───────────────────────────────
    let e = {
        let inner = {
            let t = 10;
            t * 3     // inner 블록의 tail expression → 30
        };
        inner + 5   // 바깥 블록의 tail expression → 35
    };               // e: i32 = 35

    println!("a={} b={:?} c={:?} d={} e={}",
             a, b, c, d, e);
    // a=7 b=() c=() d=42 e=35
}

함수 반환값 — 블록 규칙의 직접 적용

rust — 세미콜론 하나가 반환값을 결정 
// ✓ 올바른 함수 — tail expression 이 반환값
fn add(a: i32, b: i32) -> i32 {
    a + b   // 세미콜론 없음 → tail expression → 반환값
}

// ✗ 세미콜론 하나가 에러를 만듦
fn broken() -> i32 {
    5 + 1;  // expression statement → 블록 값 = ()
            // error[E0308]: expected `i32`, found `()`
}

// ✓ return 키워드로 조기 반환
fn safe_div(a: i32, b: i32) -> Option<i32> {
    if b == 0 {
        return None;   // 조기 반환
    }
    Some(a / b)       // tail expression → 반환값
}

// ✓ 복잡한 계산도 블록으로 정리
fn classify(n: i32) -> &'static str {
    if n < 0      { "음수" }
    else if n == 0 { "영"   }
    else           { "양수" }
    // if expression 이 tail expression — return 불필요
}

fn main() {
    println!("{}", add(3, 4));          // 7
    println!("{}", safe_div(10, 2).unwrap()); // 5
    println!("{}", classify(-3));        // 음수
}
6

match expression — 공식 문서 기준 완전 해부

공식 문서 (Reference § Match expressions): "A match expression branches on a pattern." match 는 scrutinee expression 을 각 arm 의 pattern 과 비교하고, 매칭된 arm 의 expression 값이 match 전체의 값이 됩니다.

match 의 구조 — 공식 문서 용어

공식 문서 기준 match 의 구성요소입니다.

rust — match 구조 해부 
// match <scrutinee> { arms }
//        ↑ scrutinee: 비교할 값을 생산하는 expression
//                      반드시 Sized 타입이어야 함
//                    { arms }: 하나 이상의 arm
// arm: pattern [if guard] => expression
//      ↑ pattern    ↑ match guard    ↑ arm body (expression)

let score = 85;

let grade = match score {   // scrutinee: score (i32 expression)
    90..=100 => "A",          // arm 1: 범위 패턴
    80..=89  => "B",          // arm 2
    70..=79  => "C",          // arm 3
    n if n >= 60 => "D",     // arm 4: 변수 바인딩 + guard
    _ => "F",                 // arm 5: wildcard (catch-all)
};
// grade: &str = "B"

match 타입 통일 규칙

공식 문서: 모든 arm 의 body expression 타입이 동일해야 합니다. 컴파일러가 첫 번째 arm 의 타입을 기준으로 나머지를 검사합니다.

그림 6 — match 타입 통일 규칙
rust — 타입 통일 예제 
fn main() {
    let i = 2;

    // ✓ 모든 arm 이 &str
    let s = match i {
        1 => "one",
        2 => "two",
        _ => "other",
    };  // match 타입: &str

    // ✓ 모든 arm 이 () — println! 은 () 반환
    match i {
        1 => println!("one"),
        _ => println!("other"),
    };  // match 타입: ()

    // ✓ ! (never type) 은 모든 타입과 호환
    let v = match i {
        0 => 0,
        n if n < 0 => panic!("음수 불가"),  // ! 타입 → i32 와 호환
        _ => i * 2,
    };  // match 타입: i32

    // ✗ 타입 불일치 — 컴파일 에러
    // let bad = match i {
    //     1 => 10,           // i32
    //     _ => "hello",      // &str ← 타입 불일치!
    // };  error[E0308]: match arms have incompatible types
}
그림 7 — match 평가 흐름

match exhaustiveness — 완전망라

Rust 컴파일러는 match 가 가능한 모든 경우를 처리하는지 검사합니다. 이것이 exhaustiveness 검사입니다. 모든 경우를 다루지 않으면 컴파일 에러가 발생합니다.

rust — exhaustiveness 예제 
fn main() {
    let b: bool = true;

    // ✓ bool 은 true / false 두 경우 → 모두 처리
    match b {
        true  => println!("참"),
        false => println!("거짓"),
    }

    // ✗ 에러: false 케이스가 없음
    // match b {
    //     true => println!("참"),
    // }  error[E0004]: non-exhaustive patterns: `false` not covered

    // ✓ _ wildcard 로 나머지 처리
    let n = 42;
    match n {
        1 => println!("one"),
        2 => println!("two"),
        _ => println!("other"),  // 나머지 모든 i32 처리

    // ✓ enum 은 모든 variant 처리해야 함
    enum Dir { N, S, E, W }
    let d = Dir::N;
    match d {
        Dir::N => println!("북"),
        Dir::S => println!("남"),
        Dir::E => println!("동"),
        Dir::W => println!("서"),

    }

변수 바인딩 · @ 바인딩 · 와일드카드

rust 
fn main() {
    let n = 15;

    // ─ 변수 바인딩 ──────────────────────────────────────
    match n {
        0       => println!("영"),
        x       => println!("값: {}", x),  // x 에 n 값 바인딩
    }

    // ─ @ 바인딩: 범위 체크 + 값 캡처 동시에 ────────────
    match n {
        x @ 1..=9   => println!("한 자리: {}", x),
        x @ 10..=99 => println!("두 자리: {}", x),  // 매칭: x=15
        x            => println!("세 자리+: {}", x),
    }

    // ─ 와일드카드 _ ─────────────────────────────────────
    // _ 는 값을 캡처하지 않는 "무시" 패턴
    let triple = (1, -2, 3);
    match triple {
        (0, y, z) => println!("첫째 0: {} {}", y, z),
        (x, 0, z) => println!("둘째 0: {} {}", x, z),
        (x, y, _) => println!("셋째 무시: {} {}", x, y), // 매칭
    }

    // ─ .. : 나머지 전체 무시 ────────────────────────────
    struct Point3 { x: i32, y: i32, z: i32 }
    let p = Point3 { x: 1, y: 2, z: 3 };
    match p {
        Point3 { x, .. } => println!("x = {}", x),  // y, z 무시
    }
}

fn main() {
    // ─ 구조체 분해 ──────────────────────────────────────
    struct Point { x: i32, y: i32 }
    let p = Point { x: 3, y: 0 };

    match p {
        Point { x: 0, y: 0 } => println!("원점"),
        Point { x, y: 0 }   => println!("x축: {}", x),  // 매칭
        Point { x: 0, y }   => println!("y축: {}", y),
        Point { x, y }       => println!("일반: {} {}", x, y),
    }

    // ─ 중첩 enum 분해 ───────────────────────────────────
    enum Shape {
        Circle(f64),
        Rect { w: f64, h: f64 },
        Triangle(f64, f64, f64),
    }

    let shapes = vec![
        Shape::Circle(5.0),
        Shape::Rect { w: 4.0, h: 3.0 },
        Shape::Triangle(3.0, 4.0, 5.0),
    ];

    for shape in &shapes {
        let area = match shape {
            Shape::Circle(r)         => std::f64::consts::PI * r * r,
            Shape::Rect { w, h }   => w * h,
            Shape::Triangle(a, b, c) => {
                let s = (a + b + c) / 2.0;
                (s * (s-a) * (s-b) * (s-c)).sqrt()
            },
        };
        println!("넓이: {:.2}", area);
    }
}

! (never type)

공식 문서 (Reference § Never type): "! is a type with no values. It represents the result of computations that never complete, such as loop { }, break, continue, return, or panic!()."

! 는 모든 타입의 subtype 입니다. 따라서 match arm 에서 다른 타입과 자유롭게 혼용할 수 있습니다.

never type 이 생기는 상황

rust 
fn main() {
    // ─ return 은 ! 타입 ──────────────────────────────────
    fn early(x: i32) -> i32 {
        if x < 0 { return 0; }  // return 0; → ! 타입
        x * 2
    }

    // ─ panic! 은 ! 타입 → match 에서 다른 타입과 공존 ───
    let n = 5;
    let v: i32 = match n {
        0 => 0,
        x if x < 0 => panic!("음수 불가"),  // ! → i32 로 강제 변환
        _ => n * 2,
    };
    println!("{}", v);  // 10

    // ─ loop { } 는 ! 타입 (break 없을 때) ───────────────
    // break val; 이 있으면 그 val 의 타입이 loop 의 타입
    let mut count = 0;
    let result: i32 = loop {
        count += 1;
        if count == 5 {
            break count * 2;  // break 가 값을 들고 나옴 → loop 타입 = i32
        }
    };
    println!("{}", result);  // 10

    // ─ continue / break 도 ! 타입 ────────────────────────
    for i in 0..5 {
        let x: i32 = match i % 2 {
            0 => continue,  // ! → i32 와 호환, 짝수는 건너뜀
            _ => i * 10,
        };
        println!("{}", x);  // 10, 30
    }

    // ─ diverging function ────────────────────────────────
    fn forever() -> ! {
        loop {}   // 절대 반환하지 않음 → 반환 타입 !
    }
    // forever(); // 이 줄 이후는 dead code
}

공식 문서 (Reference § If let expressions, While let loop expressions): if let 과 while let 은 하나의 패턴만 처리할 때 match 보다 간결합니다.

if let — 하나의 패턴만 처리

rust 
fn main() {
    // ─ if let 기본 ──────────────────────────────────────
    let opt: Option<i32> = Some(42);

    // match 로 쓸 때
    match opt {
        Some(v) => println!("값: {}", v),
        None    => {}
    }

    // if let 으로 동일하게
    if let Some(v) = opt {
        println!("값: {}", v);
    }

    // ─ if let + else ─────────────────────────────────────
    let cfg: Option<&str> = None;
    if let Some(val) = cfg {
        println!("설정값: {}", val);
    } else {
        println!("기본값 사용");
    }

    // ─ if let + else if let 체인 ──────────────────────────
    let result: Result<i32, &str> = Ok(10);
    if let Ok(v) = result {
        println!("성공: {}", v);
    } else if let Err(e) = result {
        println!("실패: {}", e);
    }

    // ─ enum 에 if let ─────────────────────────────────────
    enum Msg { Move(i32, i32), Stop }
    let msg = Msg::Move(5, 10);

    if let Msg::Move(x, y) = msg {
        println!("이동: {} {}", x, y);
    }
}

rust — let-else (Rust 1.65+) 
fn process(input: Option<i32>) -> i32 {
    // let-else: 패턴 매칭 안되면 else 블록 실행 (반드시 diverge)
    let Some(value) = input else {
        return 0;  // 반드시 diverge (return / break / panic 등)
    };
    // 이 아래에서 value 는 i32 로 바인딩된 상태
    value * 2
}

fn main() {
    println!("{}", process(Some(5)));  // 10
    println!("{}", process(None));      // 0
}

집약 예제 — 모든 개념 한 번에

rust 
fn evaluate(input: i32) -> String {
    // ① match 는 expression → let 에 바인딩
    let category = match input {
        i32::MIN..=-1 => "negative",
        0              => "zero",
        1..=9          => "single",
        10..=99        => "double",
        _              => "large",
    };

    // ② if expression → let 에 바인딩
    let parity = if input % 2 == 0 { "even" } else { "odd" };

    // ③ 블록 expression → let 에 바인딩
    let adjusted = {
        let base  = input.abs();
        let bonus = 10;
        base + bonus   // tail expression
    };

    // ④ Option 으로 안전하게 처리
    let opt_msg = if input == 0 { Some("영입니다") } else { None };
    let extra = match opt_msg {
        Some(s) => format!(" ({})", s),
        None    => String::new(),
    };

    // ⑤ 함수 마지막 expression = 반환값 (return 불필요)
    format!("{}{} / {} / adjusted={}",
        category, extra, parity, adjusted)
}

fn main() {
    println!("{}", evaluate(0));    // zero (영입니다) / even / adjusted=10
    println!("{}", evaluate(-7));   // negative / odd / adjusted=17
    println!("{}", evaluate(42));   // double / even / adjusted=52
    println!("{}", evaluate(100));  // large / even / adjusted=110
}
핵심 규칙 3개 — 공식 문서 기준

1.  expr               → 블록/함수의 tail expression. 그 블록/함수의 값이 됩니다.

2.  expr ;            → expression statement: 값 버려지고 ().

3.  let x = expr ;    → let statement: 값은 x 에 바인딩. ; 는 선언 종결자 (값을 버리는 역할 아님).
