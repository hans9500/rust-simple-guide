01

HashMap 이란?

출처: doc.rust-lang.org/std/collections/struct.HashMap.html

HashMap<K, V>은 키(K)와 값(V)을 쌍으로 저장하는 컬렉션입니다. 해시 함수를 이용해 키를 "버킷"에 매핑하므로, 평균 O(1)으로 조회·삽입·삭제가 가능합니다.

Rust의 HashMap은 std::collections에 속하며 프리뷰에 자동 포함되지 않으므로 반드시 use로 가져와야 합니다.

rust — use 선언
use std::collections::HashMap;

핵심 특징

특징설명
키 고유성같은 키로 삽입하면 기존 값이 덮어써지고 Some(이전값)이 반환됨
순서 없음삽입 순서가 보장되지 않음. 순서가 필요하면 BTreeMap 사용
소유권 이동삽입 시 키와 값 모두 HashMap이 소유권을 가져감 (Copy 타입은 복사됨)
동적 크기필요에 따라 자동으로 늘어남. 용량을 미리 알면 with_capacity 권장
Key 조건Eq + Hash 트레잇을 구현해야 키로 사용 가능
⚠ 소유권 주의

String처럼 Copy를 구현하지 않은 타입은 insert() 시 소유권이 이동됩니다. 삽입 후 원래 변수는 사용할 수 없습니다. 참조를 삽입하려면 라이프타임이 HashMap보다 길어야 합니다.

rust — 소유권 이동 예시
use std::collections::HashMap;

let key   = String::from("hello");
let value = String::from("world");
let mut map = HashMap::new();

map.insert(key, value);

// ❌ 컴파일 에러: key, value는 이미 map이 소유함
// println!("{}", key);

// ✅ map을 통해 접근
println!("{:?}", map);

HashMap 생성 방법 4가지

출처: doc.rust-lang.org/std/collections/struct.HashMap.html

① new() — 빈 맵 생성 (가장 일반적)

rust
use std::collections::HashMap;

// 타입을 명시하거나 나중에 추론되게 둘 수 있음
let mut map: HashMap<String, i32> = HashMap::new();
map.insert(String::from("one"), 1);

② with_capacity(n) — 초기 용량 지정

삽입할 항목 수를 미리 알고 있을 때 사용합니다. 내부 재할당 횟수를 줄여 성능을 개선합니다. 실제 용량은 요청보다 클 수 있습니다.

rust
let mut map = HashMap::with_capacity(100);
map.insert(1, "one");
println!("capacity: {}", map.capacity()); // 100 이상

③ HashMap::from([...]) — 배열 리터럴에서 생성

출처: 공식 문서 예제

rust
use std::collections::HashMap;

let solar_distance = HashMap::from([
    ("Mercury", 0.4),
    ("Venus",   0.7),
    ("Earth",   1.0),
    ("Mars",    1.5),
]);
println!("{:?}", solar_distance["Earth"]); // 1.0

④ Iterator::collect() — 이터레이터에서 수집

rust
use std::collections::HashMap;

let pairs = vec![("a", 1), ("b", 2), ("c", 3)];
let map: HashMap<&str, i32> = pairs.into_iter().collect();
println!("{:?}", map);
03

기본 연산 — 삽입 · 조회 · 수정 · 삭제

insert() — 삽입 / 덮어쓰기

키가 없으면 None 반환, 키가 이미 있으면 기존 값을 교체하고 Some(이전값) 반환합니다.

rust — 공식 문서 예제
use std::collections::HashMap;
let mut map = HashMap::new();

assert_eq!(map.insert(37, "a"), None);       // 처음 삽입 → None
assert_eq!(map.is_empty(), false);

map.insert(37, "b");
assert_eq!(map.insert(37, "c"), Some("b")); // 덮어씀 → Some("b")
assert_eq!(map[&37], "c");

get() / get_mut() — 조회

get()은 Option<&V>를 반환합니다. 키가 없으면 None이므로 안전하게 처리할 수 있습니다.

rust
use std::collections::HashMap;
let mut map = HashMap::from([("Alice", 90), ("Bob", 85)]);

// get() → Option<&V>
match map.get("Alice") {
    Some(score) => println!("Alice: {}", score),  // Alice: 90
    None        => println!("없음"),
}

// get_mut() → Option<&mut V> : 값 직접 수정
if let Some(score) = map.get_mut("Bob") {
    *score += 5;  // Bob: 85 → 90
}

// 인덱스 연산자 [] — 키가 없으면 panic!
println!("Bob: {}", map["Bob"]);  // Bob: 90
⛔ [] 연산자 주의

map["key"]는 키가 없으면 panic합니다. 키의 존재가 불확실하면 반드시 get()을 사용하세요.

contains_key() — 키 존재 확인

rust
if map.contains_key("Alice") {
    println!("Alice 데이터 있음");
}

remove() — 삭제

키를 제거하고 Option<V>를 반환합니다. 키가 없으면 None.

rust
let removed = map.remove("Alice");
println!("{:?}", removed);  // Some(90)

let none = map.remove("없는키");
println!("{:?}", none);     // None

len() / is_empty() — 크기 확인

rust
println!("항목 수: {}", map.len());
println!("비어있나: {}", map.is_empty());

HashMap 순회 방법 3가지

공식 문서에는 HashMap을 순회하는 3가지 방법이 있습니다. 소유권에 따라 구분됩니다.

방법키 타입값 타입소유권
iter() / &map&K&V빌림 — map 재사용 가능
iter_mut()&K&mut V가변 빌림 — 값 수정 가능
into_iter()KV소유권 이동 — map 소비됨
rust — 3가지 순회 비교
use std::collections::HashMap;

let mut map = HashMap::from([
    ("a", 1), ("b", 2), ("c", 3)
]);

// ① iter() — 불변 참조 순회 (&K, &V)
for (key, val) in &map {  // = map.iter()
    println!("key={} val={}", key, val);
}

// ② iter_mut() — 가변 참조 순회 (&K, &mut V)
// 공식 문서 예제: 모든 값을 2배로
for (_, val) in map.iter_mut() {
    *val *= 2;
}
// map은 이제 {"a":2, "b":4, "c":6}

// ③ into_iter() — 소유권 이동 (K, V)
for (key, val) in map {  // = map.into_iter()
    println!("소유: {}={}", key, val);
}
// map은 이 이후 사용 불가

keys() / values() — 키/값만 순회

rust
let map = HashMap::from([("x", 10), ("y", 20)]);

let keys: Vec<_>   = map.keys().collect();
let values: Vec<_> = map.values().collect();
println!("keys:   {:?}", keys);
println!("values: {:?}", values);

커스텀 struct를 키로 사용하기

출처: 공식 문서 — Viking 예제

HashMap의 키로 사용하려면 Hash + Eq + PartialEq 트레잇을 구현해야 합니다. #[derive]로 자동 구현하는 것이 가장 쉬운 방법입니다.

왜 Hash + Eq + PartialEq 가 필요한가?

Hash: 키를 해시 버킷에 배치할 때 사용. Eq: 같은 버킷 내에서 키가 동일한지 비교. PartialEq: Eq를 구현하려면 반드시 필요.

rust — 공식 문서 Viking 예제
use std::collections::HashMap;

#[derive(Hash, Eq, PartialEq, Debug)]
struct Viking {
    name:    String,
    country: String,
}

impl Viking {
    fn new(name: &str, country: &str) -> Viking {
        Viking {
            name:    name.to_string(),
            country: country.to_string(),
        }
    }
}

let mut vikings = HashMap::new();
vikings.insert(Viking::new("Einar", "Norway"),  25);
vikings.insert(Viking::new("Olaf",  "Denmark"), 24);
vikings.insert(Viking::new("Harald", "Iceland"), 12);

for (viking, health) in &vikings {
    if *health < 20 {
        println!("{} ({}) needs help!", viking.name, viking.country);
    }
}
출력
Harald (Iceland) needs help!

해시 알고리즘 — SipHash 1-3

출처: doc.rust-lang.org/std/collections/struct.HashMap.html

Rust의 HashMap은 기본적으로 SipHash 1-3 알고리즘을 사용합니다. 이는 HashDoS(해시 충돌 기반 서비스 거부) 공격에 저항하기 위해 설계되었습니다.

공식 문서가 명시하는 SipHash 1-3 특징

• 랜덤 시드를 사용해 매 실행마다 다른 해시값 생성 (예측 불가)
• 중간 크기 키에서 매우 경쟁력 있는 성능
• 정수처럼 작은 키나 긴 문자열처럼 큰 키에서는 다른 알고리즘이 더 빠를 수 있음
• 단, 다른 알고리즘들은 보통 HashDoS를 방어하지 않음

커스텀 Hasher 사용 (성능이 중요한 경우)

성능이 매우 중요하고 신뢰할 수 있는 입력만 처리한다면 BuildHasher 트레잇을 구현한 서드파티 해셔(예: FxHashMap, AHashMap)를 쓸 수 있습니다.

rust — 커스텀 Hasher 구조 (개념)
use std::collections::HashMap;
use std::hash::BuildHasherDefault;
use std::collections::hash_map::DefaultHasher;

// 기본 hasher를 명시적으로 지정하는 방법 (개념 이해용)
type MyMap<K, V> = HashMap<K, V, BuildHasherDefault<DefaultHasher>>;

Entry API — 핵심 개념

출처: doc.rust-lang.org/std/collections/hash_map/enum.Entry.html

entry(key)는 HashMap의 특정 키 위치에 대한 "뷰(view)"를 반환합니다. 이 뷰는 키가 있는지 없는지를 나타내는 열거형 Entry입니다.

rust — Entry 열거형 정의 (표준 라이브러리)
// pub enum Entry<'a, K, V>
pub enum Entry<'a, K, V> {
    Occupied(OccupiedEntry<'a, K, V>),  // 키가 존재하는 뷰
    Vacant(VacantEntry<'a, K, V>),      // 키가 없는 뷰
}
entry(key) 호출 흐름
map.entry("key")
HashMap 내부 탐색 (1회)
→
Entry (enum)
키 위치에 대한 뷰
Occupied
키가 이미 있음
기존 값 읽기/수정/삭제
|
Vacant
키가 없음
insert()로 새 값 삽입
왜 Entry API를 쓰는가? — 단 한 번의 탐색

Entry API를 사용하면 키를 한 번만 탐색합니다. contains_key()로 확인하고 get_mut()으로 다시 접근하면 두 번 탐색해 비효율적입니다.

rust — Entry 없이 vs Entry 사용 비교
use std::collections::HashMap;
let mut map: HashMap<&str, i32> = HashMap::new();

// ❌ Entry 없이 — 키를 최대 3번 탐색
if map.contains_key("score") {           // 1번째 탐색
    let v = map.get_mut("score").unwrap(); // 2번째 탐색
    *v += 1;
} else {
    map.insert("score", 1);               // 3번째 탐색
}

// ✅ Entry 사용 — 키를 딱 한 번만 탐색
*map.entry("score").or_insert(0) += 1;

공식 문서 player_stats 예제

출처: doc.rust-lang.org — Entry API 예제

rust — 공식 문서 그대로
use std::collections::HashMap;

let mut player_stats = HashMap::new();

fn random_stat_buff() -> u8 { 42 }

// 키가 없을 때만 삽입
player_stats.entry("health").or_insert(100);

// 함수로 기본값 생성 — 키가 없을 때만 함수 호출
player_stats.entry("defence").or_insert_with(random_stat_buff);

// 키가 있든 없든 가변 참조를 받아 수정
let stat = player_stats.entry("attack").or_insert(100);
*stat += random_stat_buff();

println!("{:?}", player_stats);
출력
{"health": 100, "defence": 42, "attack": 142}

Entry 메서드 완전 정리

출처: doc.rust-lang.org/std/collections/hash_map/enum.Entry.html

메서드반환 타입동작값 평가 시점
or_insert(val)&mut V없으면 val 삽입, 있으면 기존 값의 &mut V 반환항상 (즉시 평가)
or_insert_with(f)&mut V없으면 f() 호출 후 삽입없을 때만
or_insert_with_key(f)&mut V없으면 f(&key) 호출, 키를 복사 없이 전달없을 때만
or_default()&mut V없으면 Default::default() 삽입없을 때만
and_modify(f)Entry있으면 f로 수정, Entry 자체 반환 (체이닝용)있을 때만
key()&K이 Entry의 키 참조 반환—
insert_entry(val)OccupiedEntry값을 설정하고 OccupiedEntry 반환항상

or_insert_with_key — 키 기반 기본값 생성

출처: 공식 문서 예제

or_insert_with(|| ...)와 달리, 클로저가 키에 대한 참조를 받습니다. 키를 복사하거나 클론하지 않고도 키를 기반으로 값을 생성할 수 있습니다.

rust — 공식 문서 예제
use std::collections::HashMap;

let mut map: HashMap<&str, usize> = HashMap::new();

// 키가 없을 때, 키 자체를 클로저 인자로 받아 값 생성
map.entry("poneyland").or_insert_with_key(|key| key.chars().count());

assert_eq!(map["poneyland"], 9); // "poneyland".len() = 9

or_default — 타입 기본값 삽입

출처: 공식 문서 예제

rust — 공식 문서 예제
use std::collections::HashMap;

// Option<u32>의 Default는 None
let mut map: HashMap<&str, Option<u32>> = HashMap::new();
map.entry("poneyland").or_default();
assert_eq!(map["poneyland"], None);

// i32의 Default는 0, String의 Default는 ""
let mut counts: HashMap<&str, i32> = HashMap::new();
*counts.entry("hits").or_default() += 1; // 0 + 1 = 1

and_modify + or_insert 체이닝

출처: 공식 문서 예제

rust — 공식 문서 예제
use std::collections::HashMap;

let mut map: HashMap<&str, u32> = HashMap::new();

// 있으면 +1, 없으면 42로 삽입
map.entry("poneyland")
   .and_modify(|e| { *e += 1 })
   .or_insert(42);
assert_eq!(map["poneyland"], 42); // 없었으므로 42 삽입

map.entry("poneyland")
   .and_modify(|e| { *e += 1 })
   .or_insert(42);
assert_eq!(map["poneyland"], 43); // 있었으므로 +1

OccupiedEntry — 키가 존재할 때

출처: doc.rust-lang.org/std/collections/hash_map/struct.OccupiedEntry.html

OccupiedEntry는 이미 존재하는 키-값 쌍에 대한 뷰입니다. 키를 다시 탐색하지 않고도 값을 읽고 수정하고 삭제할 수 있습니다.

메서드반환설명
key()&K키에 대한 참조 반환
get()&V값에 대한 불변 참조 반환
get_mut()&mut V값에 대한 가변 참조 반환 (OccupiedEntry가 살아있는 동안)
into_mut()&'a mut V값의 가변 참조를 map 수명에 바인딩해 반환 (OccupiedEntry 소비)
insert(val)V새 값으로 교체, 이전 값 반환
remove()V키-값 제거 후 값만 반환
remove_entry()(K, V)키-값 모두 제거 후 소유권과 함께 반환

get_mut() vs into_mut() 차이

공식 문서 설명

get_mut(): OccupiedEntry에 대한 참조가 살아있는 동안만 유효한 가변 참조를 반환. Entry를 여러 번 재사용할 수 있음.
into_mut(): OccupiedEntry를 소비(move)하여 map 자체 수명에 바인딩된 가변 참조 반환. Entry를 한 번만 사용하고 끝낼 때 적합.

rust — 공식 문서 예제: get_mut() 재사용
use std::collections::HashMap;
use std::collections::hash_map::Entry;

let mut map: HashMap<&str, u32> = HashMap::new();
map.entry("poneyland").or_insert(12);
assert_eq!(map["poneyland"], 12);

if let Entry::Occupied(mut o) = map.entry("poneyland") {
    *o.get_mut() += 10;
    assert_eq!(*o.get(), 22);
    // OccupiedEntry를 여러 번 재사용 가능
    *o.get_mut() += 2;
}
assert_eq!(map["poneyland"], 24);
rust — 공식 문서 예제: into_mut()
if let Entry::Occupied(o) = map.entry("poneyland") {
    // Entry 소비 → map 수명에 바인딩된 &mut V 반환
    *o.into_mut() += 10;
}
assert_eq!(map["poneyland"], 34);

remove_entry() — 키와 값 모두 회수

rust — 공식 문서 예제
use std::collections::HashMap;
use std::collections::hash_map::Entry;

let mut map: HashMap<&str, u32> = HashMap::new();
map.entry("poneyland").or_insert(12);

if let Entry::Occupied(o) = map.entry("poneyland") {
    // (K, V) 튜플로 키와 값 모두 소유권 회수
    let (key, val) = o.remove_entry();
    println!("제거됨: key={}, val={}", key, val);
}
assert_eq!(map.contains_key("poneyland"), false);
출력
제거됨: key=poneyland, val=12

VacantEntry — 키가 없을 때

출처: doc.rust-lang.org/std/collections/hash_map/struct.VacantEntry.html

메서드반환설명
key()&K삽입하려는 키의 참조 반환
into_key()K키의 소유권을 가져옴 (Entry 소비)
insert(val)&mut V값을 삽입하고 가변 참조 반환
rust — Occupied / Vacant 직접 분기
use std::collections::HashMap;
use std::collections::hash_map::Entry;

let mut map: HashMap<&str, i32> = HashMap::new();

// ── Vacant 분기 ────────────────────────────────
match map.entry("score") {
    Entry::Occupied(mut occ) => {
        println!("기존 값: {}", occ.get());
        *occ.get_mut() += 1;
    }
    Entry::Vacant(vac) => {
        println!("없음, 삽입: key={}", vac.key());
        vac.insert(100); // 이 시점에 HashMap에 삽입됨
    }
}
println!("{:?}", map); // {"score": 100}

// ── Occupied 분기 ──────────────────────────────
match map.entry("score") {
    Entry::Occupied(mut occ) => {
        println!("기존 값: {}", occ.get()); // 100
        occ.insert(200);                       // 100 → 200
    }
    Entry::Vacant(_) => {}
}
println!("{:?}", map); // {"score": 200}
출력
없음, 삽입: key=score {"score": 100} 기존 값: 100 {"score": 200}

예제 01 — 단어 빈도 세기 (or_insert + 역참조)

가장 기본적인 Entry 패턴입니다. or_insert(0)은 &mut i32를 반환하므로 *count += 1로 역참조해 값을 수정합니다.

출처 패턴: Rust Book ch08-03

rust
use std::collections::HashMap;

fn main() {
    let text = "hello world wonderful world hello world";
    let mut map: HashMap<&str, u32> = HashMap::new();

    for word in text.split_whitespace() {
        let count = map.entry(word).or_insert(0);
        // or_insert는 &mut u32 반환 → * 로 역참조해 수정
        *count += 1;
    }

    // 결과 정렬 출력
    let mut pairs: Vec<_> = map.iter().collect();
    pairs.sort_by_key(|&(k, _)| *k);
    for (word, count) in pairs {
        println!("{:<12} : {}", word, count);
    }
}
출력
hello : 2 wonderful : 1 world : 3
예제 02 — 학생 점수 누적 (or_insert_with + Vec)

or_insert_with(Vec::new)는 키가 없을 때만 Vec::new()를 호출합니다. or_insert(vec![])는 항상 빈 Vec을 만들어 넘기므로 불필요한 할당이 생깁니다.

rust
use std::collections::HashMap;

fn main() {
    let mut scores: HashMap<String, Vec<i32>> = HashMap::new();

    let data = vec![
        ("Alice", 95), ("Bob", 80), ("Alice", 88),
        ("Charlie", 75), ("Bob", 92), ("Alice", 79),
    ];

    for (name, score) in data {
        scores
            .entry(name.to_string())
            .or_insert_with(Vec::new)  // 없을 때만 Vec 생성
            .push(score);
    }

    let mut names: Vec<_> = scores.keys().collect();
    names.sort();
    for name in names {
        let list = &scores[name];
        let avg: f64 = list.iter().sum::<i32>() as f64 / list.len() as f64;
        println!("{:<8} {:?}  평균={:.1}", name, list, avg);
    }
}
출력
Alice [95, 88, 79] 평균=87.3 Bob [80, 92] 평균=86.0 Charlie [75] 평균=75.0
예제 03 — 재고 관리 (and_modify + or_insert 체이닝)

and_modify()는 키가 존재할 때만 실행하고 Entry를 그대로 반환합니다. 이어서 or_insert()를 체이닝하면 있으면 수정, 없으면 삽입을 원자적으로 처리합니다.

rust
use std::collections::HashMap;

fn main() {
    let mut inventory: HashMap<&str, u32> = HashMap::new();

    let orders = vec![
        ("apple", 10), ("banana", 5),
        ("apple", 3),  ("orange", 8),
        ("banana", 2), ("apple", 7),
    ];

    for (item, qty) in orders {
        inventory
            .entry(item)
            .and_modify(|e| *e += qty)  // 있으면 누적
            .or_insert(qty);              // 없으면 신규 등록
    }

    let mut items: Vec<_> = inventory.iter().collect();
    items.sort_by_key(|(k, _)| *k);
    println!("=== 최종 재고 ===");
    for (item, qty) in items {
        println!("{:<8}: {}개", item, qty);
    }
}
출력
=== 최종 재고 === apple : 20개 banana : 7개 orange : 8개
예제 04 — 그룹화 (or_default + Vec 누적)

or_default()는 Vec<_>의 기본값인 빈 Vec을 자동으로 삽입합니다. 타입만 맞으면 작성이 간결합니다.

rust
use std::collections::HashMap;

fn main() {
    let students = vec![
        ("Math",    "Alice"),
        ("Science", "Bob"),
        ("Math",    "Charlie"),
        ("Science", "Diana"),
        ("History", "Eve"),
        ("Math",    "Frank"),
    ];

    let mut by_subject: HashMap<&str, Vec<&str>> = HashMap::new();

    for (subject, student) in students {
        // Vec<&str>의 Default = 빈 Vec → or_default()로 간결하게
        by_subject.entry(subject).or_default().push(student);
    }

    let mut subjects: Vec<_> = by_subject.keys().collect();
    subjects.sort();
    for subject in subjects {
        println!("{}: {:?}", subject, by_subject[subject]);
    }
}
출력
History: ["Eve"] Math: ["Alice", "Charlie", "Frank"] Science: ["Bob", "Diana"]
예제 05 — or_insert_with_key 활용

출처 패턴: 공식 문서 Entry::or_insert_with_key

키 자체를 복사하지 않고 키 기반으로 기본값을 생성합니다. 키 문자열 분석, 파싱 등에 유용합니다.

rust
use std::collections::HashMap;

fn main() {
    let mut word_lengths: HashMap<&str, usize> = HashMap::new();

    let words = vec!["rust", "programming", "hello", "rust", "world"];

    for word in words {
        // 키가 없으면 키 자체를 인자로 받아 글자 수를 기본값으로 저장

            .entry(word)
            .or_insert_with_key(|k| k.chars().count());
    }

    let mut pairs: Vec<_> = word_lengths.iter().collect();
    pairs.sort_by_key(|(k, _)| *k);
    for (word, len) in pairs {
        println!("{:<12} → 글자 수: {}", word, len);
    }
}

예제 06 — 조건부 삭제 (remove_entry)

값이 임계값 미만인 항목을 순회하며 제거하는 패턴입니다. remove_entry()는 키와 값을 모두 소유권과 함께 반환합니다.

rust
use std::collections::HashMap;
use std::collections::hash_map::Entry;

fn main() {
    let mut scores: HashMap<String, i32> = HashMap::from([
        ("Alice".to_string(),   95),
        ("Bob".to_string(),     42),
        ("Charlie".to_string(), 38),
        ("Diana".to_string(),   88),
    ]);

    let threshold = 50;
    let keys_to_check: Vec<String> = scores.keys().cloned().collect();
    let mut removed = vec![];

    for key in keys_to_check {
        if let Entry::Occupied(o) = scores.entry(key) {
            if *o.get() < threshold {
                let (name, score) = o.remove_entry(); // (K, V) 반환
                removed.push((name, score));
            }
        }
    }

    println!("탈락자:");
    for (name, score) in &removed {
        println!("  {} ({}점)", name, score);
    }
    println!("잔여: {:?}", scores);
}
출력
탈락자: Bob (42점) Charlie (38점) 잔여: {"Alice": 95, "Diana": 88}
예제 07 — 커스텀 struct 키 활용

#[derive(Hash, Eq, PartialEq)]를 붙인 struct를 키로 사용하는 패턴입니다.

rust
use std::collections::HashMap;

#[derive(Hash, Eq, PartialEq, Debug, Clone)]
struct Point { x: i32, y: i32 }

fn main() {
    let mut grid: HashMap<Point, &str> = HashMap::new();

    grid.insert(Point { x: 0, y: 0 }, "origin");
    grid.insert(Point { x: 1, y: 0 }, "east");
    grid.insert(Point { x: 0, y: 1 }, "north");

    // Entry로 없으면 삽입
    grid.entry(Point { x: 0, y: 0 }).or_insert("already set");
    grid.entry(Point { x: 2, y: 2 }).or_insert("diagonal");

    let probe = Point { x: 0, y: 0 };
    println!("({},{}) = {:?}", probe.x, probe.y, grid.get(&probe));

    for (pt, label) in &grid {
        println!("({},{}) → {}", pt.x, pt.y, label);
    }
}
출력
(0,0) = Some("origin") (0,0) → origin (1,0) → east (0,1) → north (2,2) → diagonal
예제 08 — 중첩 HashMap (도시별 월별 기온)

HashMap의 값이 다시 HashMap인 중첩 구조입니다. or_insert_with(HashMap::new)로 내부 맵을 지연 생성합니다.

rust
use std::collections::HashMap;

fn main() {
    // HashMap<도시, HashMap<월, 기온>>
    let mut temp_db: HashMap<&str, HashMap<u32, f32>> = HashMap::new();

    let records = vec![
        ("Seoul", 1,  -2.5),
        ("Seoul", 7,  25.6),
        ("Busan", 1,  3.4),
        ("Busan", 7,  27.1),
        ("Seoul", 4,  13.2),
    ];

    for (city, month, temp) in records {
        temp_db
            .entry(city)
            .or_insert_with(HashMap::new)  // 도시가 없으면 빈 맵 생성
            .insert(month, temp);         // 내부 맵에 월별 기온 삽입
    }

    let mut cities: Vec<_> = temp_db.keys().collect();
    cities.sort();
    for city in cities {
        let months = &temp_db[city];
        let mut ms: Vec<_> = months.iter().collect();
        ms.sort_by_key(|(m, _)| *m);
        print!("{}:", city);
        for (month, temp) in ms {
            print!("  {}월={:.1}°C", month, temp);
        }
        println!();
    }
}

&&i32 — 이중 참조가 생기는 이유

출처: doc.rust-lang.org/std/iter/trait.Iterator.html#method.filter

filter() 클로저 안에서 타입이 예상과 다르게 이중 참조(&&i32)가 되는 현상은 공식 문서가 직접 "possibly confusing situation"이라고 인정한 내용입니다.

filter() 의 공식 시그니처

rust — Iterator trait 공식 정의
fn filter<P>(self, predicate: P) -> Filter<Self, P>
where
    P: FnMut(&Self::Item) -> bool
//          ^^^^^^^^^^^^
//          클로저는 Self::Item 을 참조(&)로 받는다
공식 문서 원문 (doc.rust-lang.org)

"Because the closure passed to filter() takes a reference, and many iterators iterate over references, this leads to a possibly confusing situation, where the type of the closure is a double reference"

참조가 두 겹이 되는 과정

공식 문서 코드 예제 3가지

rust — 공식 문서 (std::iter::Iterator::filter) 그대로
let s = &[0, 1, 2];

// 방법 1: x = &&i32, ** 로 두 번 역참조 (공식 문서: "needs two *s!")
let mut iter = s.iter().filter(|x| **x > 1);
assert_eq!(iter.next(), Some(&2));
assert_eq!(iter.next(), None);

// 방법 2: &x 패턴으로 한 겹 벗기기 → x = &i32
let mut iter = s.iter().filter(|&x| *x > 1);
assert_eq!(iter.next(), Some(&2));

// 방법 3: &&x 패턴으로 두 겹 모두 벗기기 → x = i32
let mut iter = s.iter().filter(|&&x| x > 1);
assert_eq!(iter.next(), Some(&2));

클로저 인자의 & 는 타입 선언이 아니라 패턴 분해

핵심 정리

|&x| 또는 |&&x| 에서의 &는 타입을 선언하는 것이 아니라 참조를 분해(destructuring)하는 패턴입니다. |&x| ...는 |x| { let &x = x; ... }와 동일합니다.

표현클로저 x 타입공식 문서 여부
|x| **x > 1&&i32 → **로 두 번 역참조✅ 공식 문서 예제
|&x| *x > 1&i32 (패턴 분해)✅ 공식 문서 예제
|&&x| x > 1i32 (패턴 분해)✅ 공식 문서 예제
|x| x > &1컴파일 오류 (&&i32 vs &i32 불일치)❌ 공식 문서에 없음
|x| *x == &1PartialEq 우연히 동작, 비권장❌ 공식 문서에 없음
