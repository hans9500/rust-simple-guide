Rust 수명(Lifetime) 완전 가이드

The Rust Programming Language 공식문서 기반 · 할루시네이션 없음

1. 메모리 구조 — 스택과 힙

Rust의 모든 값은 스택(Stack) 또는 힙(Heap) 중 하나에 저장됩니다. 이 구분이 소유권과 수명 시스템의 근간입니다.

스택 (Stack)

  • 컴파일 타임에 크기가 확정된 값
  • LIFO 구조 · 할당/해제 매우 빠름
  • 함수 종료 시 자동 해제
  • 예: i32 bool f64 char &T [T;N]

힙 (Heap)

  • 런타임에 크기가 결정 · 동적 할당
  • 포인터를 통해 접근
  • 소유자가 사라지면 drop() 자동 호출
  • 예: String Vec<T> Box<T>
스택(좌) → 힙/바이너리(우) 포인터 흐름과 소유권 관계

2. String vs &str

두 타입의 차이를 정확히 이해하는 것이 수명 이해의 출발점입니다.

항목&strString
소유권없음있음
가변성불변가변 가능
저장 위치바이너리 또는 힙의 일부힙
크기(스택)16 bytes (ptr+len)24 bytes (ptr+len+cap)
수명참조 대상에 묶임소유자가 결정
drop없음스코프 종료 시 자동
리터럴'static 수명해당 없음

📌 공식문서 Chapter 8.2

"A String is made up of three parts: a pointer to the memory that holds the contents of the string, a length, and a capacity. This group of data is stored on the stack."

UTF-8과 인덱싱 주의

let s = "안녕하세요";
for c in s.chars() { print!("{c} "); }  // 안 녕 하 세 요
println!("chars={} bytes={}", s.chars().count(), s.len());  // 5, 15
// let bad = &s[0..1]; // ❌ 런타임 패닉! 잘못된 UTF-8 경계

3. 소유권(Ownership)

소유권은 GC 없이 메모리 안전성을 보장하는 핵심 메커니즘입니다.

소유권 3대 규칙 (공식문서 Chapter 4.1)

1. 각 값은 소유자(owner)가 있다.

2. 한 번에 소유자는 하나뿐이다.

3. 소유자가 스코프를 벗어나면 값은 drop된다.

Move / Copy / Clone 세 가지 값 전달 방식

4. 차용(Borrowing) 규칙

참조를 생성하는 것을 차용이라 합니다. borrow checker가 컴파일 타임에 모든 규칙을 검증합니다.

차용 규칙 (공식문서 Chapter 4.2)

1. 불변 참조(&T)는 몇 개든 동시에 가질 수 있다.

2. 또는 가변 참조(&mut T) 하나만 가질 수 있다.

3. 참조는 항상 유효해야 한다 (댕글링 포인터 불가).

차용 규칙 3가지 — borrow checker가 컴파일 타임에 보장

5. 수명(Lifetime) — 참조의 유효 범위

수명은 참조가 유효한 스코프를 나타내는 컴파일 타임 정보입니다. 런타임 동작을 바꾸지 않습니다.

수명 생략 규칙 3가지 (공식문서 Chapter 10.3)

규칙 1: 각 참조 입력 파라미터는 각자의 수명을 가진다.

  fn foo(x: &str, y: &str) → fn foo<'a,'b>(x: &'a str, y: &'b str)

규칙 2: 입력 수명이 정확히 하나라면 출력에도 같은 수명이 적용된다.

  fn foo(x: &str) -> &str → fn foo<'a>(x: &'a str) -> &'a str

규칙 3: &self/&mut self가 있으면 출력 수명은 self의 수명이 된다.

세 규칙 적용 후에도 수명을 알 수 없으면 → 명시적 어노테이션 필요

'static 두 가지 의미 (혼동 주의)

① 참조 수명: 프로그램 전체 실행 기간 동안 유효 → 문자열 리터럴은 바이너리에 저장

② 트레이트 바운드: 타입이 non-static 참조를 포함하지 않음 (dangling 위험 없음)

예제 01 스코프와 Drop — LIFO 순서


struct Res { name: String }
impl Drop for Res {
    fn drop(&mut self) { println!("[DROP] {}", self.name); }
}
fn main() {
    let _a = Res { name: "A".into() };
    {
        let _b = Res { name: "B".into() };
        let _c = Res { name: "C".into() };
        println!("A B C 살아있음");
    } // → [DROP] C, [DROP] B  (LIFO!)
    println!("A만 남음");
} // → [DROP] A

예제 04 함수와 소유권 이동


fn takes_ownership(s: String) { println!("got: {s}"); }
fn gives_ownership() -> String { String::from("given") }
fn take_and_give(s: String) -> String { s }
fn main() {
    let s1 = String::from("hello");
    takes_ownership(s1);
    // println!("{s1}"); // ❌ E0382
    let n = 42;
    let _ = n;
    println!("n still: {n}");  // ✓ Copy 타입
    let s2 = gives_ownership();
    let s3 = take_and_give(s2);
    println!("{s3}");
}

예제 06 Clone과 Copy 비교


#[derive(Debug, Clone, Copy)]  // Copy: Clone도 필수
struct Point { x: f64, y: f64 }

#[derive(Debug, Clone)]         // Copy 불가: String이 Drop 구현
struct Person { name: String, age: u32 }

fn main() {
    let p1 = Point { x:1.0, y:2.0 };
    let p2 = p1;                    // Copy: p1도 유효
    println!("p1={p1:?}, p2={p2:?}");

    let alice = Person { name:"Alice".into(), age:30 };
    let alice2 = alice.clone();   // 힙 재할당
    println!("{alice:?}, {alice2:?}");
}






예제 07 불변 참조 · Deref Coercion


fn print_info(s: &str) {
    println!("chars={} bytes={}", s.chars().count(), s.len());
}
fn main() {
    let s = String::from("Hello 안녕");
    let r1 = &s;
    let r2 = &s;  // 불변 참조 동시에 여럿 OK
    println!("{r1} / {r2}");
    print_info(&s);        // &String → &str (Deref coercion)

    print_info(&b);         // Box → &String → &str

예제 08 가변 참조 · NLL · split_at_mut


fn add_suffix(s: &mut String, sfx: &str) { s.push_str(sfx); }
fn main() {
    let mut s = String::from("hello");
    add_suffix(&mut s, " world");
    let r1 = &s;
    let r2 = &s;
    println!("{r1} {r2}");  // ← NLL 종료
    let r3 = &mut s;        // ✓ r1,r2 NLL 종료 후 가능
    r3.push('!');
    let mut v = vec![1,2,3,4,5];
    for x in v.iter_mut() { *x *= 2; }
    let (l,r) = v.split_at_mut(2);
    l[0]=100; r[0]=999;
    println!("{v:?}");
}

예제 10 수명 명시 기본


fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() >= y.len() { x } else { y }
}
struct Excerpt<'a> { part: &'a str }
impl<'a> Excerpt<'a> {
    fn announce(&self, msg: &str) -> &str {
        println!("[공지] {msg}"); self.part
    }
}
fn main() {
    let s1 = String::from("long string");
    {
        let s2 = String::from("xyz");
        println!("{}", longest(&s1, &s2));
    }
    let novel = String::from("옛날. 두 번째.");
    let ex = Excerpt { part: novel.split('.').next().unwrap() };
    println!("{}", ex.announce("발췌"));
}

심화 01 다중 수명 파라미터


'a: 'b는 'a가 'b보다 오래 산다는 수명 바운드입니다.


fn first_of<'a,'b>(x: &'a str, _y: &'b str) -> &'a str { x }
struct Context<'src,'tok> { source: &'src str, token: &'tok str }
impl<'src,'tok> Context<'src,'tok> {
    fn new(s: &'src str, t: &'tok str) -> Self { Self{source:s,token:t} }
    fn source(&self) -> &'src str { self.source }
    fn token(&self) -> &'tok str  { self.token }
}
fn longer<'a:'b,'b>(x: &'a str, y: &'b str) -> &'b str {
    if x.len() > y.len() { x } else { y }
}
fn main() {
    let src = String::from("fn main() {}");
    let result;
    {
        let tok = String::from("fn");
        let ctx = Context::new(&src, &tok);
        println!("src={}, tok={}", ctx.source(), ctx.token());
        result = first_of(&src, &tok);
    } // tok drop, result는 src 수명이라 OK
    println!("result: {result}");
    println!("longer: {}", longer(&src, "x"));
}






심화 02 수명을 가진 커스텀 Iterator


Item 타입에 수명을 붙이면 반환 슬라이스가 원본에 묶여 zero-copy 이터레이션이 가능합니다.


struct WordIter<'a> { text: &'a str, pos: usize }
impl<'a> WordIter<'a> {
    fn new(t: &'a str) -> Self { Self{text:t,pos:0} }
}
impl<'a> Iterator for WordIter<'a> {
    type Item = &'a str;
    fn next(&mut self) -> Option<Self::Item> {
        if self.pos >= self.text.len() { return None; }
        let rest = &self.text[self.pos..];
        match rest.find(' ') {
            Some(i) => { let w=&rest[..i]; self.pos+=i+1; Some(w) }
            None    => { self.pos=self.text.len(); Some(rest) }
        }
    }
}
fn main() {
    let text = String::from("the quick brown fox");
    for w in WordIter::new(&text) { print!("[{w}]"); }
    println!();
    let long: Vec<&str> = WordIter::new(&text).filter(|w| w.len()>3).collect();
    println!("{long:?}");
}






심화 03 Rc / RefCell / Weak


Rc: 단일 스레드 참조 카운팅. RefCell: 런타임 차용 검사. Weak: 순환 참조 방지.


use std::cell::RefCell;
use std::rc::{Rc,Weak};
struct Node {
    val:i32,
    children:RefCell<Vec<Rc<Node>>>,
    parent:RefCell<Weak<Node>>,
}
fn main() {
    let a = Rc::new("shared".to_string());
    let b = Rc::clone(&a);
    println!("count={}",Rc::strong_count(&a));  // 2
    drop(b);
    let shared = Rc::new(RefCell::new(vec![0]));
    Rc::clone(&shared).borrow_mut().push(1);
    Rc::clone(&shared).borrow_mut().push(2);
    println!("{:?}",shared.borrow());  // [0,1,2]
    let root = Rc::new(Node{val:1,children:RefCell::new(vec![]),parent:RefCell::new(Weak::new())});
    let child = Rc::new(Node{val:2,children:RefCell::new(vec![]),parent:RefCell::new(Rc::downgrade(&root))});
    root.children.borrow_mut().push(Rc::clone(&child));
    println!("strong={} weak={}",Rc::strong_count(&root),Rc::weak_count(&root));
}






심화 04 클로저와 수명 — Fn/FnMut/FnOnce


move로 소유권을 캡처하면 'static을 충족합니다. 참조를 캡처하면 클로저 수명이 참조에 묶입니다.


fn make_adder(x:i32) -> impl Fn(i32)->i32 { move |y| x+y }
fn make_checker<'a>(list: &'a [i32]) -> impl Fn(i32)->bool + 'a {
    move |x| list.contains(&x)
}
fn apply_once<F:FnOnce()->String>(f:F)->String { f() }
fn make_greeting(pre:String)->Box<dyn Fn(&str)->String> {
    Box::new(move |name| format!("{pre}, {name}!"))
}
fn main() {
    let add5 = make_adder(5);
    println!("3+5={}",add5(3));
    let primes = vec![2,3,5,7,11];
    let is_prime = make_checker(&primes);
    println!("5={}, 6={}",is_prime(5),is_prime(6));
    let msg = String::from("한번만");
    println!("{}",apply_once(move ||msg));
    let hi = make_greeting("Hello".into());
    println!("{}",hi("Rust"));
}






심화 05 dyn Trait + impl Trait + 수명


trait Summarize { fn summary(&self)->String; }
struct Article{title:String,body:String}
struct Tweet{user:String,text:String}
impl Summarize for Article{
    fn summary(&self)->String{format!("[기사] {}: {}...",self.title,&self.body[..self.body.len().min(20)])}
}
impl Summarize for Tweet{
    fn summary(&self)->String{format!("@{}: {}",self.user,self.text)}
}
fn notify(x:&impl Summarize){println!("알림: {}",x.summary());}
fn make(k:&str)->Box<dyn Summarize>{
    if k=="a"{Box::new(Article{title:"뉴스".into(),body:"내용".into()})}
    else{Box::new(Tweet{user:"rust".into(),text:"New!".into()})}
}
fn print_all<'a>(items:&[&'a dyn Summarize]){
    for(i,x)initems.iter().enumerate(){println!("[{}] {}",i+1,x.summary());}
}
fn main(){
    let items:Vec<Box<dyn Summarize>>=vec![make("a"),make("t")];
    let refs:Vec<&dyn Summarize>=items.iter().map(|b|b.as_ref()).collect();
    print_all(&refs);
    let t=Tweet{user:"bob".into(),text:"rocks".into()};
    notify(&t);
}

심화 07 Zero-copy Parser


struct Parser<'a>{input:&'a str,pos:usize}
impl<'a> Parser<'a>{
    fn new(s:&'a str)->Self{Self{input:s,pos:0}}
    fn peek(&self)->Option<char>{self.input[self.pos..].chars().next()}
    fn skip_ws(&mut self){
        while self.peek().map_or(false,|c|c.is_whitespace()){
            self.pos+=self.peek().unwrap().len_utf8();
        }
    }
    fn parse_word(&mut self)->Option<&'a str>{
        self.skip_ws();
        let start=self.pos;
        while self.peek().map_or(false,|c|!c.is_whitespace()){
            self.pos+=self.peek().unwrap().len_utf8();
        }
        if self.pos>start{Some(&self.input[start..self.pos])}else{None}
    }
    fn parse_all(&mut self)->Vec<&'a str>{
        let mut v=vec![];
        while let Some(w)=self.parse_word(){v.push(w);}v
    }
}
fn parse_kv<'a>(s:&'a str)->Vec<(&'a str,&'a str)>{
    s.lines().filter_map(|l|{
        let mut p=l.splitn(2,'=');
        Some((p.next()?.trim(),p.next()?.trim()))
    }).collect()
}
fn main(){
    let text=String::from("  hello   world  rust  ");

심화 08 Arena 패턴


struct NodeId(usize);
struct Node{val:i32,children:Vec<NodeId>,parent:Option<usize>}
struct Arena{nodes:Vec<Node>}
impl Arena{
    fn new()->Self{Arena{nodes:vec![]}}
    fn add(&mut self,val:i32,par:Option<usize>)->NodeId{
        let id=self.nodes.len();
        self.nodes.push(Node{val,children:vec![],parent:par});
        NodeId(id)
    }
    fn link(&mut self,par:usize,child:NodeId){self.nodes[par].children.push(child);}
    fn dfs(&self,id:usize,d:usize){
        println!("{}{}","  ".repeat(d),self.nodes[id].val);
        for NodeId(c)in&self.nodes[id].children{self.dfs(*c,d+1);}
    }
}
fn main(){
    let mut a=Arena::new();
    let NodeId(r)=a.add(1,None);
    let c1=a.add(2,Some(r));
    let c2=a.add(3,Some(r));
    a.link(r,c1);a.link(r,c2);
    a.dfs(r,0);
}






심화 09 스레드 + 'static + thread::scope


thread::spawn은 Send+'static을 요구합니다. thread::scope(Rust 1.63+)는 non-static 참조를 안전하게 공유합니다.


use std::sync::{Arc,Mutex};
use std::thread;
fn main(){
    let msg=String::from("hello");
    thread::spawn(move ||println!("{msg}")).join().unwrap();
    let counter=Arc::new(Mutex::new(0i32));
    let mut handles=vec![];
    for i in 0..5{
        let c=Arc::clone(&counter);
        handles.push(thread::spawn(move ||{*c.lock().unwrap()+=i;}));
    }
    for h in handles{h.join().unwrap();}
    println!("합계: {}",*counter.lock().unwrap());
    let data=vec![1,2,3,4,5];
    let mut result=0i32;
    thread::scope(|s|{
        s.spawn(||{result=data.iter().sum();});
    });
    println!("scoped sum: {result}");
}






심화 10 Pin — 자기참조 구조체와 async/await 원리


Pin은 데이터를 메모리에 고정합니다. async fn은 내부적으로 자기참조 Future로 컴파일되며 Pin이 필요합니다.


use std::pin::Pin;
use std::marker::PhantomPinned;
struct SelfRef{
    data:String,
    ptr:*const u8,
    _pin:PhantomPinned,
}
impl SelfRef{
    fn new(s:&str)->Pin<Box<Self>>{
        let mut p=Box::pin(SelfRef{data:String::from(s),ptr:std::ptr::null(),_pin:PhantomPinned});
        let ptr=p.data.as_ptr();
        unsafe{p.as_mut().get_unchecked_mut().ptr=ptr;}
        p
    }
    fn data(self:Pin<&Self>)->&str{&self.data}
    fn valid(self:Pin<&Self>)->bool{self.data.as_ptr()==self.ptr}
}
fn main(){
    let s=SelfRef::new("pinned");
    println!("data={}, valid={}",s.as_ref().data(),s.as_ref().valid());
}

❌ E0382 — use of moved value


let s1=String::from("hi");
let s2=s1;
println!("{s1}"); // ❌


원인: String은 Move 타입. s1→s2 소유권 이전.
해결: s1.clone() 또는 &s1






❌ E0505 — cannot move out of borrow


let r=&s;
let s2=s; // ❌ r이 빌리는 중


원인: 차용 중인 값 이동 불가.
해결: r 사용 종료 후 이동 또는 clone






❌ E0502 — cannot borrow as mutable


let r1=&s;
let r2=&mut s; // ❌
println!("{r1}");


원인: 불변 참조 유효 중 가변 참조 불가.
해결: r1 마지막 사용 후 &mut (NLL)






❌ E0499 — 가변 참조 두 개 동시


let r1=&mut s;
let r2=&mut s; // ❌


원인: 가변 참조는 한 번에 하나.
해결: r1 사용 완료 후 r2 생성






❌ E0106 — missing lifetime specifier


fn foo(x:&str,y:&str)->&str{
    x // 어느 수명? 모호
}


원인: 생략 규칙으로 추론 불가.
해결: fn foo<'a>(x:&'a str,..)->&'a str






❌ E0597 — does not live long enough


let r;
{let x=String::from("x");
 r=&x;} // x drop → 댕글링!


원인: x가 r보다 먼저 drop.
해결: 같은 스코프 또는 String 반환






❌ E0716 — temporary dropped while borrowed


let s=&String::from("hi");
// 임시값 즉시 drop → 댕글링!


원인: 임시 String이 문장 끝에서 drop.
해결: let owned=...; let s=&owned;






❌ E0310 — 'static 바운드 미충족


fn needs(s:&'static str){..}
let owned=String::from("hi");
needs(&owned); // ❌


원인: owned는 'static이 아님.
